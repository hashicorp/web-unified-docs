# AGENTS.md — Agent (coding) mode

This file provides guidance to agents when writing or modifying code in this repository.

## Must-follow patterns

### Result type — no exceptions
All fallible operations return `Result<T, E>` from `#utils/result`. Use `Ok(value)` / `Err(error?)`. Never `throw` for expected error conditions. Check `.ok` before accessing `.value`.

### Named exports only
`eslint-plugin-no-barrel-files` + `no-restricted-exports` are both active. Do not create index re-export files. Do not use `export default`.

### Arrow function body style
Always write `() => { return x }`, never `() => x` — `arrow-body-style: always` is enforced as a warning.

### Required braces
All control flow (`if`, `else`, `for`, `while`) must have curly braces — `curly: 'error'`.

### Copyright header
Every new `.ts`/`.mjs`/`.js` file needs:
```js
/**
 * Copyright IBM Corp. 2024, 2026
 * SPDX-License-Identifier: BUSL-1.1
 */
```

### Path aliases — always use, never relative across packages
| Alias | Maps to |
|---|---|
| `#utils/*` | `app/utils/*` |
| `#api/*` | `app/api/*` |
| `#productConfig.mjs` | `productConfig.mjs` |
| `#scriptUtils/*` | `scripts/utils/*` |

Exception: relative paths are acceptable within the same directory (e.g. `./result` inside `app/utils/`).

## Testing

- Test files sit next to source (`foo.ts` → `foo.test.ts`), never in a separate `__tests__/` folder.
- Use `mockRequest()` from `#utils/mockRequest` to invoke Next.js API route handlers in tests.
- Use `vi.mock('fs')` + `vi.mock('fs/promises')` to activate the `memfs`-backed `__mocks__/fs.mjs`. Call `vol.reset()` in `beforeEach`.
- `vi.mock('#api/versionMetadata.json', () => ({ default: {} }))` and `vi.mock('#api/docsPathsAllVersions.json', () => ({ default: {} }))` are boilerplate needed in most route/util tests because those JSONs are imported at module load time.
- When `process.env` is read at module load time, set it with `vi.hoisted(() => { process.env.X = '...' })` **before** the import that reads it.
- Shared test data lives in `__fixtures__/` — check there before creating new mock data.

## Prebuild outputs — never edit manually
`app/api/versionMetadata.json` and `app/api/docsPathsAllVersions.json` are generated by `npm run prebuild`. Do not edit them. If tests need them, mock with `vi.mock()`.

## Adding a new product
Register it in `productConfig.mjs` (`PRODUCT_CONFIG`). That is the single source of truth for all product metadata, content directories, version handling, and API inclusion.
