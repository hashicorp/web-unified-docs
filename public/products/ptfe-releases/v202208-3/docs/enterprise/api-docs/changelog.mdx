# API Changelog

Keep track of changes to the API for Terraform Cloud and Terraform Enterprise.

### 2022-07-26

-   Updated the [Run status list](/enterprise/api-docs/run#run-states) with `fetching`, `queuing`, `pre_plan_running` and `pre_plan_completed`
-   Update [Run Tasks](/enterprise/api-docs/run-tasks.mdx) to include the new `stages` attribute when attaching or updating workspace tasks.
-   Updated [Run Tasks Integration](/enterprise/api-docs/run-tasks/run-tasks-integration) to specify the different request payloads for different stages.

### 2022-06-23

-   Added the [Assessments](/enterprise/api-docs/assessments).
-   Updated [Workspace](/enterprise/api-docs/workspaces#create-a-workspace) and
    [Notification Configurations](/enterprise/api-docs/notification-configurations#notification-triggers) to account for assessments.
-   Added new query parameter(s) to [List Runs endpoint](/enterprise/api-docs/run#list-runs-in-a-workspace).

### 2022-06-21

-   Updated [Admin Organizations](/enterprise/api-docs/admin/organizations) endpoints with new `workspace-limit` setting. This is available in Terraform Enterprise v202207-1 and later.
-   Updated [List Agent Pools](/enterprise/api-docs/agents/#list-agent-pools) to accept a filter parameter `filter[allowed_workspaces][name]` so that agent pools can be filtered by name of an associated workspace. The given workspace must be allowed to use the agent pool. Refer to [Scoping Agent Pools to Specific Workspaces](/enterprise/agents#scope-an-agent-pool-to-specific-workspaces).
-   Added new `organization-scoped` attribute and `allowed-workspaces` relationship to the request/response body of the below endpoints. This is available in Terraform Enterprise v202207-1 and later.
    -   [Show an Agent Pool](/enterprise/api-docs/agents/#show-an-agent-pool)
    -   [Create an Agent Pool](/enterprise/api-docs/agents/#create-an-agent-pool)
    -   [Update an Agent Pool](/enterprise/api-docs/agents/#update-an-agent-pool)

### 2022-06-17

-   Updated [Creating a Run Task](/enterprise/workspaces/settings/run-tasks#creating-a-run-task) section to include the new description information for the run task to be configured.
-   Update [Run Tasks](/enterprise/api-docs/run-tasks.mdx) to include the new description attribute.

### 2022-06-09

-   Updated [List Agent Pools](/enterprise/api-docs/agents/#list-agent-pools) to accept a search parameter `q`, so that agent pools can be searched by `name`. This is available in Terraform Enterprise v202207-1 and later.
-   Fixed [List Workspaces](/enterprise/api-docs/workspaces#list-workspaces) to add missing `search[tags]` query parameter.
-   Updated [List Workspaces](/enterprise/api-docs/workspaces#list-workspaces) to add new `search[exclude_tags]` query parameter. This is available in Terraform Enterprise v202207-1 and later.

### 2022-05-11

-   Updated Run Tasks permission to the following endpoints:
    -   [Organizations](/enterprise/api-docs/organizations#list-organizations).
    -   [Team Access](/enterprise/api-docs/team-access#list-team-access-to-a-workspace).
    -   [Teams](/enterprise/api-docs/teams#list-teams).

### 2022-05-04

-   Updated [Feature Sets](https://www.terraform.io/enterprise/api-docs/feature-sets#list-feature-sets) to add new `run-tasks` attribute.

### 2022-05-03

-   Added Run Tasks permission to the following endpoints:
    -   [Organizations](/enterprise/api-docs/organizations#list-organizations)
    -   [Workspaces](/enterprise/api-docs/workspaces#show-workspace)

### 2022-04-29

-   Updated [Run Tasks Integration](/enterprise/api-docs/run-tasks/run-tasks-integration) to specify the allowed `status` attribute values.
-   Updated [Organization Memberships](/enterprise/api-docs/organization-memberships#query-parameters) to add new `filter[email]` query parameter.

### 2021-06-8

-   Updated [Registry Module APIs](/enterprise/api-docs/modules).
    -   added `registry_name` scoped APIs.
    -   added `organization_name` scoped APIs.
    -   added [Module List API](/enterprise/api-docs/modules#list-registry-modules-for-an-organization).
    -   updated [Module Delete APIs](/enterprise/api-docs/modules#delete-a-module) (see deprecation note below).
    -   **CLOUD**: added public registry module related APIs.
-   **DEPRECATION**: The following [Registry Module APIs](/enterprise/api-docs/modules) have been replaced with newer apis and will be removed in the future.
    -   The following endpoints to delete modules are replaced with [Module Delete APIs](/enterprise/api-docs/modules#delete-a-module)
        -   `POST /registry-modules/actions/delete/:organization_name/:name/:provider/:version` replaced with `DELETE /organizations/:organization_name/registry-modules/:registry_name/:namespace/:name/:provider/:version`
        -   `POST /registry-modules/actions/delete/:organization_name/:name/:provider` replaced with `DELETE /organizations/:organization_name/registry-modules/:registry_name/:namespace/:name/:provider`
        -   `POST /registry-modules/actions/delete/:organization_name/:name` replaced with `DELETE /organizations/:organization_name/registry-modules/:registry_name/:namespace/:name`
    -   `POST /registry-modules` replaced with [`POST /organizations/:organization_name/registry-modules/vcs`](/enterprise/api-docs/modules#publish-a-private-module-from-a-vcs)
    -   `POST /registry-modules/:organization_name/:name/:provider/versions` replaced with [`POST /organizations/:organization_name/registry-modules/:registry_name/:namespace/:name/:provider/versions`](/enterprise/api-docs/modules#create-a-module-version)
    -   `GET /registry-modules/show/:organization_name/:name/:provider` replaced with [`GET /organizations/:organization_name/registry-modules/:registry_name/:namespace/:name/:provider`](/enterprise/api-docs/modules#get-a-module)
