# Telemetry

Telemetry in terraform-mcp-server is implemented with OpenTelemetry metrics and is enabled only when OTEL_METRICS_ENABLED=true. In HTTP mode, the server creates an OTel meter provider, exports metrics over OTLP/HTTP to OTEL_METRICS_ENDPOINT, and flushes them on the interval set by OTEL_METRICS_EXPORT_INTERVAL. The service identity is added through OTEL_METRICS_SERVICE_NAME and OTEL_METRICS_SERVICE_VERSION.

## Two kinds of metrics are collected. 
First, standard HTTP server metrics are added by wrapping the HTTP mux with otelhttp.NewHandler(...). This emits:

http.server.request.body.size
http.server.response.body.size
http.server.request.duration

Second, the MCP server records custom tool metrics around tool execution using MCP hooks (BeforeCallTool / AfterCallTool). These emit:

mcp_tool_calls_total
mcp_tool_errors_total
mcp_tool_duration_seconds

## How the data flows 

terraform-mcp-server generates the metrics, exports them to the configured OTLP endpoint, and terraform-mcp-server-remote can then receive or forward that telemetry to Datadog through its own collector/forwarding setup. In short: this repo produces OTel metrics; the remote repo is responsible for routing them to Datadog.

## Important scope note

This metrics setup is currently tied to the streamable HTTP server path. It is not the default stdio path.
