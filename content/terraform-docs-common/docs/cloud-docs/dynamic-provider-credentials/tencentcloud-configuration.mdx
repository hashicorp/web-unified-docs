# Use dynamic credentials with the Tencent Cloud provider

You can use HCP Terraform's native OpenID Connect integration with Tencent Cloud to get [dynamic credentials](/terraform/cloud-docs/dynamic-provider-credentials) for the Tencent Cloud provider in your HCP Terraform runs. Configuring the integration requires the following steps:

1. **[Configure Tencent Cloud](#configure-tencent-cloud):** Set up a trust configuration between Tencent Cloud and HCP Terraform. Then, you must create a Tencent Cloud CAM role for your HCP Terraform workspaces.
2. **[Configure HCP Terraform](#configure-hcp-terraform):** Add environment variables to the HCP Terraform workspaces where you want to use Dynamic Credentials.

<!-- BEGIN: TFC:only name:stacks-tfe -->

If you are configuring a Stack, refer to [Authenticate a Stack](/terraform/language/stacks/deploy/authenticate) to learn about the workflow of authenticating your Stack deployments with dynamic credentials.

<!-- END: TFC:only name:stacks-tfe -->

Once you complete the setup, HCP Terraform automatically authenticates to Tencent Cloud during each run. The Tencent Cloud provider authentication is valid for the length of the plan or apply.

## Requirements for agents

If you are self-hosting [HCP Terraform agents](/terraform/cloud-docs/agents), ensure your agents use [v1.17.0](/terraform/cloud-docs/agents/changelog) or above. To use the latest dynamic credentials features, [upgrade your agents to the latest version](/terraform/cloud-docs/agents/changelog).

## Configure Tencent Cloud

You must enable and configure an OIDC identity provider and an accompanying CAM role on Tencent Cloud. These instructions use the Tencent Cloud console.

### Prerequisites

- A [Tencent Cloud](https://www.tencentcloud.com/) account
- Access to the [Tencent Cloud CAM (Cloud Access Management)](https://console.tencentcloud.com/cam) console
- The [`tencentcloudstack/tencentcloud`](https://registry.terraform.io/providers/tencentcloudstack/tencentcloud/latest) Terraform provider

### Create an OIDC Identity Provider

1. Navigate to the [Tencent Cloud CAM Console](https://console.tencentcloud.com/cam/idp).
2. Select **Identity Providers** and then **Create Identity Provider**.
3. Choose **OIDC** as the provider type.
4. Configure the following settings:
   - **Provider URL**: Set to the address of HCP Terraform (`https://app.terraform.io` without a trailing slash), or the URL of your Terraform Enterprise instance.
   - **Client ID (Audience)**: Set to `tencentcloud.workload.identity`, or the value of `TFC_TENCENTCLOUD_WORKLOAD_IDENTITY_AUDIENCE` if you configure a custom audience.

### Create a CAM Role

You must create a CAM role that HCP Terraform assumes during runs.

1. Navigate to [Tencent Cloud CAM Roles](https://console.tencentcloud.com/cam/role).
2. Select **Create Role** and choose **Identity Provider** as the role entity.
3. Select **OIDC** as the identity provider type.
4. Select the OIDC identity provider you created in the previous step.
5. Configure the trust conditions. At minimum, verify the audience (`aud`) claim to prevent unauthorized access from other HCP Terraform organizations:
   - **Audience**: `tencentcloud.workload.identity` (or your custom audience)
   - **Subject (`sub`)**: You can restrict access to specific organizations, projects, and workspaces. The HCP Terraform token subject format is `organization:ORG_NAME:project:PROJECT_NAME:workspace:WORKSPACE_NAME:run_phase:PHASE`.
6. Attach the permission policies required for your infrastructure operations to the role.
7. Note the **Role ARN** after creation. The format is `qcs::cam::uin/ACCOUNT_ID:roleName/ROLE_NAME`.

!> **Warning**: You should always check, at minimum, the audience and the name of the organization in order to prevent unauthorized access from other HCP Terraform organizations.

You can set these as workspace variables, or if you'd like to share one Tencent Cloud role across multiple workspaces, you can use a variable set. When you configure dynamic provider credentials with multiple provider configurations of the same type, use either a default variable or a tagged alias variable name for each provider configuration. ## Configure the Tencent Cloud Provider

When using dynamic credentials, HCP Terraform writes a web identity token file containing the OIDC JWT and provides the token file path and role ARN through a Terraform variable. The Tencent Cloud provider uses the `assume_role_with_web_identity` block to read the token and perform the STS `AssumeRoleWithWebIdentity` call directly.

Make sure that you are passing a value for the `region` argument into the provider configuration block or setting the `TENCENTCLOUD_REGION` variable in your workspace.

Make sure that you are not using any of the other credential arguments (such as `secret_id`, `secret_key`, or `security_token`) in the provider configuration, as these settings may interfere with dynamic provider credentials.

### Required Terraform Variable

Add the following variable to your Terraform configuration. HCP Terraform automatically supplies values for this variable during runs, providing the token file path and role ARN for each configuration.

```hcl
variable "tfc_tencentcloud_dynamic_credentials" {
  description = "Object containing Tencent Cloud dynamic credentials configuration"
  type = object({
    default = object({
      token_file = string
      role_arn   = string
    })
    aliases = map(object({
      token_file = string
      role_arn   = string
    }))
  })
}
```

### Single provider instance

#### Example Usage

```hcl
provider "tencentcloud" {
  region = "ap-guangzhou"

  assume_role_with_web_identity {
    web_identity_token_file = var.tfc_tencentcloud_dynamic_credentials.default.token_file
    role_arn                = var.tfc_tencentcloud_dynamic_credentials.default.role_arn
    provider_id             = "app.terraform.io"
  }
}
```

### Specifying Multiple Configurations

~> **Important:** If you are self-hosting [HCP Terraform agents](/terraform/cloud-docs/agents), ensure your agents use [v1.17.0](/terraform/cloud-docs/agents/changelog) or above. To use the latest dynamic credentials features, [upgrade your agents to the latest version](/terraform/cloud-docs/agents/changelog).

You can add additional variables to handle multiple distinct Tencent Cloud setups, enabling you to use multiple [provider aliases](/terraform/language/providers/configuration#alias-multiple-provider-configurations) within the same workspace. You can configure each set of credentials independently, or use default values by configuring the variables prefixed with `TFC_DEFAULT_`.

For more details, see [Specifying Multiple Configurations](/terraform/cloud-docs/dynamic-provider-credentials/specifying-multiple-configurations).

#### Example Usage

```hcl
provider "tencentcloud" {
  region = "ap-guangzhou"

  assume_role_with_web_identity {
    web_identity_token_file = var.tfc_tencentcloud_dynamic_credentials.default.token_file
    role_arn                = var.tfc_tencentcloud_dynamic_credentials.default.role_arn
    provider_id             = "app.terraform.io"
  }
}

provider "tencentcloud" {
  alias  = "ALIAS1"
  region = "ap-shanghai"

  assume_role_with_web_identity {
    web_identity_token_file = var.tfc_tencentcloud_dynamic_credentials.aliases["ALIAS1"].token_file
    role_arn                = var.tfc_tencentcloud_dynamic_credentials.aliases["ALIAS1"].role_arn
    provider_id             = "app.terraform.io"
  }
}
```

## Troubleshooting

### Invalid trust relationship

If you receive an error indicating that the trust relationship is invalid, verify the following:
- The OIDC identity provider URL in Tencent Cloud CAM matches the address of your HCP Terraform instance (e.g., `https://app.terraform.io`).
- The audience (Client ID) configured in the OIDC identity provider matches `tencentcloud.workload.identity` or the custom audience set via `TFC_TENCENTCLOUD_WORKLOAD_IDENTITY_AUDIENCE`.

### Incorrect Role ARN format

The CAM role ARN must follow the format `qcs::cam::uin/ACCOUNT_ID:roleName/ROLE_NAME`. Verify that the ARN you specified in `TFC_TENCENTCLOUD_RUN_ROLE_ARN` (or plan/apply variants) matches the role ARN shown in the Tencent Cloud CAM console.

### Missing permissions

If Terraform operations fail with permission errors, verify that the CAM role has the required permission policies attached. Check the role's permission policies in the [Tencent Cloud CAM console](https://console.tencentcloud.com/cam/role).
