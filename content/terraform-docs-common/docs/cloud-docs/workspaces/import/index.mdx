# Search and import resources

HCP Terraform lets you search your existing infrastructure for resources and import them into your workspace without writing HCL queries. This feature helps you discover unmanaged resources across cloud providers, making it easier to bring existing infrastructure under Terraform management.

## Introduction

When you inherit infrastructure or manage environments with unknown resources, identifying and importing unmanaged resources can be a time-consuming manual process. The search and import feature provides a visual interface to build queries so that HCP Terraform can discover resources across your cloud providers without requiring you to manually write Terraform configuration to import the resources.

## Workflows

You can search for resources to import into your Terraform workspaces either by using the HCP Terraform UI to build queries, or by creating queries within your Terraform configuration.

### Query builder workflow

To import resources into Terraform using the query builder workflow, you will:

1. Define queries: Use the query builder in the HCP Terraform UI to target resource types and use filters to narrow results by tags, regions, or other attributes.
1. Run the queries: You can run queries in the HCP Terraform UI or run the Terraform CLI on your local workstation.
1. Review search results: HCP Terraform shows the management status for resources it finds.
1. Generate code: HCP Terraform generates `import` and `resource` blocks for the resources it discovers.
1. Apply the configuration: Copy the generated code to your Terraform configuration and run it to finish importing the resources.

For detailed instructions on running queries and importing resources, refer to the [query documentation](/terraform/cloud-docs/workspaces/import/query).

### Configuration-driven import workflow

To import resources into Terraform using the configuration-driven workflow, you will:

1. Define queries: Add `list` blocks to your Terraform configuration. If you are using the VCS-driven workflow, commit this change and push it to the repository associated with your workspace in HCP Terraform. Refer to [Import resources in bulk](/terraform/language/import/bulk) for more information.
1. Run the queries: You can run queries in the HCP Terraform UI or run the Terraform CLI on your local workstation.
1. Review search results: HCP Terraform shows the management status for resources it finds.
1. Generate code: HCP Terraform generates `import` and `resource` blocks for the resources it discovers.
1. Apply the configuration: Copy the generated code to your Terraform configuration and run it to finish importing the resources.

For detailed instructions on how to import resources by writing queries using list blocks in your Terraform configuration, refer to the [Import existing resources to state](/terraform/cloud-docs/workspaces/import/configuration) documentation.

### Constraints and limitations

- You must enable Terraform 1.14 or newer for your workspace to access the search and import feature.
- Search queries require appropriate cloud provider credentials configured in your workspace.
- If you use HCP Terraform agents, you must enable the `query` operation when starting the agent pool.
- Resources managed by other workspaces are identified only when those workspaces use Terraform v1.12 or newer.
