# Diagnostics API reference

The diagnostics endpoint performs comprehensive health diagnostics of all Terraform Enterprise subsystems. It returns detailed status information for each component, including warnings and errors. You can customize the timeout and filter specific checks.

## Perform diagnostics

This endpoint runs diagnostic checks on Terraform Enterprise subsystems. Authentication is required to use this endpoint.

`GET /api/v1/diagnostics`

### Query parameters

| Parameter | Type | Description |
| --- | --- | --- |
| `timeout` | integer | Optional. Diagnostic check timeout in seconds. Valid range is 1-300. Default is 30. |
| `check` | array of strings | Optional. Filter checks to run by group or specific checks. Format: `{group_name}` or `{group_name.check_name}`. Multiple values are comma-separated in the URL, for example: `?check=redis,database.valid_config`. If omitted, all checks run. |

### Sample request

```shell
curl \
  --header "Authorization: Bearer $TOKEN" \
  --request GET \
  "https://tfe.example.com:8443/api/v1/diagnostics?timeout=60&check=redis,database"
```

### Sample response

```json
{
  "status": "completed",
  "timestamp": "2026-02-03T12:00:00Z",
  "duration_ms": 4500,
  "checks": {
    "redis": {
      "status": "ok",
      "checks": {
        "valid_config": {
          "status": "ok",
          "message": "Redis configuration is valid"
        },
        "connectivity": {
          "status": "ok",
          "message": "Successfully connected to Redis"
        }
      }
    },
    "database": {
      "status": "warning",
      "checks": {
        "valid_config": {
          "status": "ok",
          "message": "Database configuration is valid"
        },
        "connectivity": {
          "status": "warning",
          "message": "Database connection slow, response time: 1500ms"
        }
      }
    }
  }
}
```

### Response attributes

| Attribute | Type | Description |
| --- | --- | --- |
| `status` | string | Overall status of the diagnostics: `completed`, `partial`, or `failed` |
| `timestamp` | datetime | When the diagnostics completed |
| `duration_ms` | integer | Time taken to run diagnostics in milliseconds |
| `checks` | object | Detailed results for each check group |
| `checks.{group_name}` | object | Status for a specific check group |
| `checks.{group_name}.status` | string | Status of the group: `ok`, `warning`, or `error` |
| `checks.{group_name}.checks` | object | Individual checks within the group |
| `checks.{group_name}.checks.{check_name}` | object | Details for a specific check |
| `checks.{group_name}.checks.{check_name}.status` | string | Status of the check: `ok`, `warning`, or `error` |
| `checks.{group_name}.checks.{check_name}.message` | string | Descriptive message for the check result |

### Response codes

| Status | Response | Reason |
| --- | --- | --- |
| [200][] | [JSON API document][] | Diagnostics completed successfully |
| [400][] | [JSON API error object][] | Invalid request parameters, such as timeout out of range |
| [406][] | [JSON API error object][] | Only application/json is supported |
| [429][] | [JSON API error object][] | Another diagnostic check is already running |
| [503][] | [JSON API document][] | One or more checks failed |
