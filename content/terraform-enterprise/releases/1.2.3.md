## Known Issues
1. Terraform Enterprise readiness endpoint `api/v1/nodes/readiness` respond with an HTTP `503` status when a node is in a draining state. This can cause readiness probes in cloud environments to treat the node as unhealthy and remove it from service, even though the node is draining and intentionally not accepting new requests. As a workaround, use the `/_health_check` endpoint instead of the readiness endpoint.
