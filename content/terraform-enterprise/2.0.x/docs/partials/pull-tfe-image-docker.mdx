<Tab heading="HashiCorp Registry" group="hashicorp">

1. Connect to the host instance.

1. Log in to the Terraform Enterprise container image registry.

   ```shell-session
   $ cat <PATH_TO_HASHICORP_LICENSE_FILE> |  docker login --username terraform images.releases.hashicorp.com --password-stdin
   ```

1. Pull the Terraform Enterprise image from the registry.  Refer to [Terraform Enterprise Releases](/terraform/enterprise/releases) for a list of versions and replace `<VERSION>` with the version you wish to install. `latest` is not a valid tag.

   ```shell-session
   $ docker pull images.releases.hashicorp.com/hashicorp/terraform-enterprise:<VERSION>
   ```

<Tab heading="IBM Entitled Registry" group="ibm-reg">

To log in to the IBM Entitled Registry, you must obtain a container registry access key from IBM Passport Advantage Online (PAO). The PAO portal provides the following credentials:
- Installation key: This installation key is equivalent to the HashiCorp license and is configured through the `TFE_LICENSE` environment variable. However, as opposed to the HashiCorp license, the installation key cannot authenticate with any container registry.
- Access key: A credential for authenticating with the IBM Entitled Registry at `cp.icr.io` when pulling the Terraform Enterprise container image.

To obtain the container registry access key:

1. Log in to the [Passport Advantage website](https://www.ibm.com/software/passportadvantage).
1. Under **Software download**, select **Download Software**.
1. Locate **IBM Terraform Self-Managed Premium** in your list of software downloads, then click **View Product**.
1. Continue to the specific product version.
1. Select the **Container Install** tab.
1. Under **Active access keys**, click **Add new key** to generate a new container registry access key for the IBM Entitled Registry. IBM Passport Advantage Online allows a maximum of five active access keys.

To download the image:

1. Connect to the host instance.

1. Log in to the IBM Entitled Registry with the container registry access key.

   ```shell-session
   $ echo "YOUR_ACCESS_KEY" | docker login cp.icr.io --username cp --password-stdin
   ```

1. Pull the Terraform Enterprise image from the registry. Refer to [Terraform Enterprise Releases](/terraform/enterprise/releases) for a list of versions and replace `<VERSION>` with the version you wish to install. `latest` is not a valid tag.

   ```shell-session
   $ docker pull cp.icr.io/cp/ibm-terraform/ibm-terraform:<VERSION>
   ```
