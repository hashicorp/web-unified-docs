Recognized group names (matched case-insensitively, surrounding whitespace ignored):

- `license`
- `tls`
- `disk`
- `database`
- `redis`
- `cloud`
- `archivist` (only relevant when `TFE_OBJECT_STORAGE_TYPE=s3`)
- `explorer_database` (only relevant when an explorer database is configured)

The following groups always hard-fail and cannot be ignored:

- `config`
- `upgrade`

Unknown values in the list are silently ignored. Diagnostic-only checks (`atlas`, `vault`, `task-worker`, `fluent-bit`) do not gate startup, so listing them has no effect. The `preupgrade-check` tool and runtime diagnostics are not affected by this setting; they continue to report failures normally.
