# Rate Limit configuration entry reference

The `rate-limit` configuration entry lets you dynamically control the rate of read and write rpc requests across your Consul cluster. Use this entry to protect your Consul cluster from excessive traffic without restarting agents.

Unlike the static per-server global limiter configured through agent configuration, this config entry is cluster-wide, hot-reloadable, and stored in Raft. Changes to this entry propagate to all servers in 50–100ms.

<EnterpriseAlert>

This feature requires Consul Enterprise.

</EnterpriseAlert>

## Configuration Model

The following list outlines field hierarchy, data types, and requirements.

- [`Kind`](#kind) - Must be set to `rate-limit`.
- [`Name`](#name) - Must be set to `global`.
- [`Config`](#config) - Block that defines the rate limiting behavior.
  - [`ReadRate`](#config-readrate) - Maximum read requests per second.
  - [`WriteRate`](#config-writerate) - Maximum write requests per second.
  - [`Priority`](#config-priority) - Determines whether the global config entry takes precedence over per-server config.
  - [`ExcludeEndpoints`](#config-excludeendpoints) - RPC methods that bypass rate limiting.

## Configuration Reference

### `Kind`

Specifies the type of configuration entry to create. Must be set to `rate-limit`.

- Default: None
- This field is required.
- Data type: String

### `Name`

Specifies the name of the configuration entry. Must be set to `global` to ensure only one active rate-limit configuration exists per cluster.

- Default: None
- This field is required.
- Data type: String

### `Config`

#### `Config.ReadRate`

Specifies the maximum number of read requests per second. Set to `0` to block all read operations.

- Default: Infinite
- Data type: Float

#### `Config.WriteRate`

Specifies the maximum number of write requests per second. Set to `0` to block all write operations.

- Default: Infinite
- Data type: Float

#### `Config.Priority`

Determines whether the global config entry takes precedence over per-server agent configuration. When set to `true`, the config entry rate limits override per-server settings. When set to `false`, per-server configuration takes precedence and the global config entry rate limits are not enforced.

- Default: `false`
- Data type: Boolean

#### `Config.ExcludeEndpoints`

Specifies a list of RPC method names that bypass rate limiting.

To retrieve a list of all available RPC method names, call the `GET /v1/internal/rpc/methods` API endpoint.

- Default: `[]`
- Data type: List of strings

## CLI Usage

Write the configuration entry to Consul.

```shell-session
$ consul config write emergency-rate-limit.hcl
```

Read the current rate-limit configuration.

```shell-session
$ consul config read -kind rate-limit -name global
```

Delete the rate-limit configuration.

```shell-session
$ consul config delete -kind rate-limit -name global
```

## Example

The following example limits read and write operations to 100 requests per second, enables priority mode to override per-server settings, excludes the `Health.Check`.

```hcl
Kind = "rate-limit"
Name = "global"

Config {
  ReadRate            = 100
  WriteRate           = 100
  Priority            = true
  ExcludeEndpoints    = ["Health.Check"]
}
```

```json
{
  "Kind": "rate-limit",
  "Name": "global",
  "Config": {
    "priority": true,
    "readRate": 100,
    "writeRate": 100,
    "excludeEndpoints": ["Health.Check"]
  }
}
```

```yaml
apiVersion: consul.hashicorp.com/v1alpha1
kind: RateLimit
metadata:
  name: global
spec:
  config:
    readRate: 100
    writeRate: 100
    priority: true
    excludeEndpoints:
      - "Health.Check"
```      

## ACL Requirements

The `rate-limit` configuration entry uses operator-level ACL permissions.

| Operation      | Required ACL         |
| -------------- | -------------------- |
| Read config    | `operator = "read"`  |
| Create/Update  | `operator = "write"` |
| Delete         | `operator = "write"` |

## Kubernetes

### Admin partitions and ACL environments

When admin partitions and ACLs are both enabled, you must provide a dedicated operator-level ACL token for the global config rate limiter controller. Without this token, reconciliation may fail even when `consul config write` succeeds.

Create a Kubernetes secret that holds the token.

```yaml
apiVersion: v1
kind: Secret
metadata:
  name: consul-global-config-acl-token
  namespace: <namespace>
type: Opaque
stringData:
  token: <operator-write-token>
```

Reference the secret in your Helm values.

```yaml
connectInject:
  globalConfigACLToken:
    secretName: consul-global-config-acl-token
    secretKey: token
```
