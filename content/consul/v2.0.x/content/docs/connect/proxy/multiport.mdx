# Multiport services

In a Consul service mesh, services can be configured to use multiple ports. This allows you to run different types of traffic through the same service while still benefiting from Consul's service discovery and traffic management features. For example, a web application might have one port for HTTP traffic and another for gRPC traffic. With Consul Connect, you can configure your services to use multiple ports while still leveraging the benefits of service mesh features such as secure communication, traffic routing, and observability.

## Configuring multiport services

You configure a multiport service in Consul Connect by specifying the ports in the service registration. Each port should have a unique name, port number, and whether it is the default port. Here's an example of the aforementioned web application with both HTTP and gRPC ports defined in HCL and JSON formats:

<CodeTabs tabs={[ "HCL", "JSON" ]}>

<CodeBlockConfig filename="multiport-service.hcl" highlight="5-13">

```hcl
service {
  name = "api"
  id   = "api-1"
  ports = [
    {
      name = "http"
      port = 8080
      default = true
    },
    {
      name = "grpc"
      port = 9090
    }
  ]
}
```

</CodeBlockConfig>

<CodeBlockConfig filename="multiport-service.json" highlight="5-13">

```json
{
  "name": "api",
  "id": "api-1",
  "ports": [
    {
      "name": "http",
      "port": 8080,
      "default": true
    },
    {
      "name": "grpc",
      "port": 9090
    }
  ]
}
```

</CodeBlockConfig>

## Addressing multiport services

### Non-Transparent proxy

In Non-Transparent proxy mode, downstream services explicitly configure upstreams by defining three items - a service `destination_name`, the `local_bind_port`, and specifically to the multiport use case - a `destination_port`. This allows the downstream service to specify which port on the upstream service it wants to connect to, and the proxy will handle routing the traffic accordingly. Check out these details further in the [upstream configuration reference](docs/reference/proxy/connect-proxy#upstream-configuration-reference).

An example configuration for a non-transparent proxy service with multiport upstream services is shown below. In this example, the `web` service connecting to `localhost:9191` will be rerouted to the single port `db:9191` service. Meanwhile, the `web` service connecting to `localhost:9192` will be rerouted to the multiport destination of `api:8080`. Finally, the `web` service connecting to `localhost:9193` will be rerouted to the multiport destination of `api:9090`.

<CodeTabs tabs={[ "HCL", "JSON" ]}>

<CodeBlockConfig language="hcl" highlight="12-21">

```hcl
service {
  name = "web"
  port = 8080
  connect {
    sidecar_service {
      proxy {
        upstreams = [
          {
            destination_name = "db"
            local_bind_port = 9191
          },
          {
            destination_name = "api"
            destination_port = "http"
            local_bind_port = 9192
          },
          {
            destination_name = "api"
            destination_port = "grpc"
            local_bind_port = 9193
          }
        ]
        config {
          handshake_timeout_ms = 1000
        }
      }
    }
  }
}
```

</CodeBlockConfig>

<CodeBlockConfig language="json" highlight="12-21">


### Transparent proxy

In Transparent Proxy mode, Consul uses virtual DNS to resolve the correct port for their upstream service without needing to specify the port in their configuration.

The virtual DNS entries are generated based on the service name and port name, following the naming convention `<port-name>.<service-name>.virtual.consul`. Consul's internal DNS returns a Virtual IP (VIP) unique to that specific service-port combination. For example, if you have a service named `api` with two ports named `http` and `grpc`, the virtual DNS entries would be `http.api.virtual.consul` and `grpc.api.virtual.consul`.

## Multiport functionality in Consul services

The following functionality has partial support for Consul Multiport:

- Mesh gateways
- Proxy and Service defaults
- Consul DNS

It is important to note that multiport in WAN Federation is not supported.
