# YDB configuration for Vault backend storage

The YDB storage backend is used to persist Vault's data in
[YDB](https://ydb.tech/), a distributed SQL database. The backend supports the
standard physical storage interface, transactional updates, and optional
high-availability locking backed by YDB coordination nodes.

- **High Availability** – the YDB storage backend supports high availability
  when `ha_enabled = "true"`.

- **Community Supported** – the YDB storage backend is supported by the
  community. While it has undergone review by HashiCorp employees, they may not
  be as knowledgeable about the technology. If you encounter problems with it,
  you may be referred to the original author.

```hcl
storage "ydb" {
  dsn   = "grpc://127.0.0.1:2136/local"
  table = "vault_kv"
}
```

The backend automatically creates the data table if it does not already exist.

## `ydb` authentication

The YDB backend uses the official YDB Go SDK. Configuration values may also be
provided through environment variables; when both are set, environment
variables take precedence.

Supported authentication methods:

- Access token using `token` or `VAULT_YDB_TOKEN`
- Service account key file using `service_account_key_file` or
  `VAULT_YDB_SA_KEYFILE`
- Service account key contents using `service_account_key` or
  `VAULT_YDB_SA_KEY`
- Static credentials using `static_credentials_user` /
  `static_credentials_password` or the corresponding
  `VAULT_YDB_STATIC_CREDENTIALS_*` variables
- Metadata credentials using `metadata_auth` or `VAULT_YDB_METADATA_AUTH`
- Anonymous credentials using `anonymous_credentials` or
  `VAULT_YDB_ANONYMOUS_CREDENTIALS`

The backend also honors standard `YDB_*` SDK environment variables as a
fallback if no Vault-specific authentication settings are configured.

## `ydb` parameters

- `dsn` `(string: <required>)` - Specifies the YDB connection string to use.
  This may also be set using the `VAULT_YDB_DSN` environment variable.

- `table` `(string: "vault_kv")` - Specifies the YDB table name to use for
  Vault data. This may also be set using the `VAULT_YDB_TABLE` environment
  variable.

- `token` `(string: "")` - Specifies an access token for YDB authentication.

- `service_account_key_file` `(string: "")` - Specifies the path to a Yandex
  Cloud service account key file.

- `service_account_key` `(string: "")` - Specifies the contents of a Yandex
  Cloud service account key.

- `static_credentials_user` `(string: "")` - Specifies the username for static
  YDB credentials.

- `static_credentials_password` `(string: "")` - Specifies the password for
  static YDB credentials.

- `metadata_auth` `(string: "false")` - Enables metadata-based credentials.

- `anonymous_credentials` `(string: "false")` - Enables anonymous authentication.

- `internal_ca` `(string: "false")` - Enables Yandex Cloud internal CA support.

- `ha_enabled` `(string: "false")` - Enables Vault HA locking through YDB
  coordination nodes.

- `ha_coordination_node` `(string: "<table>_ha")` - Specifies the YDB
  coordination node path to use for HA locking. If the value is not an absolute
  YDB path, Vault resolves it relative to the database in the configured DSN.

- `transaction_max_entries` `(int: 63)` - Specifies the maximum number of
  operations Vault should batch into a single backend transaction.

- `transaction_max_size` `(int: 131072)` - Specifies the maximum combined raw
  key and value size, in bytes, Vault should batch into a single backend
  transaction.

## `ydb` examples

### Basic configuration

```hcl
storage "ydb" {
  dsn   = "grpc://127.0.0.1:2136/local"
  table = "vault_kv"
}
```

### High availability

```hcl
storage "ydb" {
  dsn                  = "grpc://127.0.0.1:2136/local"
  table                = "vault_kv"
  ha_enabled           = "true"
  ha_coordination_node = "/local/vault_kv_ha"
}
```

### Service account key file authentication

```hcl
storage "ydb" {
  dsn                      = "grpcs://ydb.example.net:2135/local"
  table                    = "vault_kv"
  service_account_key_file = "/etc/vault/ydb-sa.json"
  internal_ca              = "true"
}
```

### Custom transaction limits

```hcl
storage "ydb" {
  dsn                     = "grpc://127.0.0.1:2136/local"
  table                   = "vault_kv"
  transaction_max_entries = 63
  transaction_max_size    = 131072
}
```

## `ydb` notes

- Table names are automatically escaped before being interpolated into YDB
  queries.
- The backend creates the storage table automatically if it is missing.
- HA lock state is stored in a YDB coordination node. The coordination node is
  created lazily on first lock use when HA is enabled.
- The default transaction limits are intentionally conservative and match
  Vault's historic transaction batching defaults.
