# `/sys/policies/rotation/`

The `/sys/policies/rotation/` endpoints are used to manage rotation policies in Vault.
Not all plugins use rotation policies, so check the documentation for the plugin you
are using for compatibility.

~> Rotation policies are only available in Vault Enterprise version 2.0+.

See [Rotation Policies](/vault/docs/concepts/rotation-policies) for details of how rotation policies work
as well as the syntax of the policies themselves.

## Create/Update rotation policy

This endpoint adds a new or updates an existing rotation policy. Once a policy is updated,
changes may not be immediately reflected to all associated rotation entries. Please refer to the
[rotation policy usage](/vault/docs/concepts/rotation-policies#usage) docs for more details.

| Method | Path                           |
| :----- | :----------------------------- |
| `POST` | `/sys/policies/rotation/:name` |

### Parameters

- `name` `(string: <required>)` – Specifies the name of the rotation policy to create.
  This is specified as part of the request URL.

- `policy` `(string: <required>)` - Specifies the rotation policy document. This must be
  valid JSON-string format. See [Rotation Policy Syntax](/vault/docs/concepts/rotation-policies#rotation-policy-syntax)
  for details on rotation policy definitions.

### Sample payload

```json
{
  "policy":  "{\n    \"max_retry_cycles\": 3,\n    \"max_retries_per_cycle\": 3\n  }"
}
```

### Sample request

**cURL:**

```shell
$ curl \
    --header "X-Vault-Token: ..." \
    --request POST \
    --data @payload.json \
    http://127.0.0.1:8200/v1/sys/policies/rotation/my-policy
```

**Vault CLI:**

```shell
$ cat policy.json
{
  "max_retry_cycles": 3,
  "max_retries_per_cycle": 3
}

$ vault write sys/policies/rotation/my-policy policy=@policy.json
```

## Read rotation policy

This endpoint retrieves information about the named rotation policy.

| Method | Path                           |
| :----- | :----------------------------- |
| `GET`  | `/sys/policies/rotation/:name` |

### Parameters

- `name` `(string: <required>)` – Specifies the name of the rotation policy to retrieve.
  This is specified as part of the request URL.

### Sample request

```shell
$ curl \
    --header "X-Vault-Token: ..." \
    http://127.0.0.1:8200/v1/sys/policies/rotation/my-policy
```

### Sample response

```json
{
    "max_retries_per_cycle": 3,
    "max_retry_cycles": 3
  }
```

## Delete rotation policy

This endpoint deletes the rotation policy with the given name. This does not check if any
secret engines are using it prior to deletion, so you should ensure that any engines that
are utilizing this rotation policy are changed to a different policy (or to that engines'
default behavior).

| Method   | Path                           |
| :------- | :----------------------------- |
| `DELETE` | `/sys/policies/rotation/:name` |

### Parameters

- `name` `(string: <required>)` – Specifies the name of the rotation policy to delete.
  This is specified as part of the request URL.

### Sample request

```shell
$ curl \
    --header "X-Vault-Token: ..." \
    --request DELETE
    http://127.0.0.1:8200/v1/sys/policies/rotation/my-policy
```
