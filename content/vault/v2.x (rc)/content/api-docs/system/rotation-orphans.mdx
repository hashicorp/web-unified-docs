# `/sys/rotation/orphans`

The `/sys/rotation/orphans` endpoints are used to manage orphaned automated rotation entries in Vault.

~> Orphaning is enabled on all plugins using the Rotation Manager and on Vault Enterprise version 2.0+.

See [Orphaning](/vault/docs/concepts/rotation-policies#orphaning) for details of how the Rotation Manager
orphans failing rotation entries.

## Read a list of orphans

This endpoint returns a list of all orphaned rotation entries in the Rotation Manager.
This endpoint can be used for observability into broken/failing rotations.

| Method | Path                           |
| :----- | :----------------------------- |
| `GET`  | `/sys/rotation/orphans/` |

### Sample request

```shell
$ curl \
    --header "X-Vault-Token: ..." \
    http://127.0.0.1:8200/v1/sys/rotation/orphans
```

### Sample response

```json
{
    "orphans": ["ldap/static-role/mary", "ldap/static-role/alice"]
}
```
