### PKI role allow_token_displayname parameter ((#pki-role-allow-token-displayname))

| Announced | Expected end of support | Expected removal |
| :-------: | :---------------------: | :--------------: |
| APR 2025  | APR 2026                | APR 2026         |

The `allow_token_displayname` parameter in PKI roles is deprecated because
token display names include the issuing authentication method making it unsuitable
as a base domain for certificate issuance.

The `allow_token_displayname` parameter defaults to `false` and when `true`
allows certificate name validation to match the requesting token’s display name 
and, when `allow_subdomains=true`, subdomains of that display name.


This parameter has not been writable since Vault version 0.4.0 or later. 
It remains for backward compatibility only and is scheduled for removal in
a later Vault release.

If you currently have `allow_token_displayname` set to `true`, we recommend that you
use alternative role constraints such as `allowed_domains`, `allow_bare_domains`,
`allow_subdomains`, and `allow_glob_domains`.
