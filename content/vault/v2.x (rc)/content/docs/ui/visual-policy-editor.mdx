# Visual Policy Editor in the Vault GUI

The Visual Policy Editor helps you create ACL policies with a guided interface
that generates formatted [HashiCorp Configuration Language (HCL)](https://github.com/hashicorp/hcl/) policy syntax.

## Generate policy flyout 

@include 'alerts/enterprise-only.mdx'

The following secrets engines support the **Generate policy** flyout:

@include 'ui/mounts-with-policy-flyout.mdx'

Use the context-aware flyout to build policies for specific workflows.
When you open the **Generate policy** flyout on a supported page, Vault
prepopulates it with API paths for that view.
You can review those paths, choose capabilities, and save the result as an ACL policy.

## Before you start

- **You must have Vault version 2.0.0 or later.**
- **You must have the appropriate permissions**:
  - To **create ACL policies**, you must have `create` permission for the [`/sys/policies/acl`](/vault/api-docs/system/policies#create-update-acl-policy) endpoint.

## Generate a policy for a KV v2 secret

1. Sign in to the Vault GUI.
1. Navigate to a KV v2 secrets engine that contains secrets.
1. Select a secret to generate a policy for.
1. Click **Generate policy** in the page header. The **Generate policy** action may be inside the **Manage** dropdown.
1. Review prepopulated paths and select capabilities.

![visual policy editor context aware flyout](/img/gui/visual-policy-editor-flyout.png)

## Create an ACL policy using the Visual Policy Editor

1. Sign in to the Vault GUI.
1. Open **Access control** and select **ACL policies**.
1. Click **Create ACL Policy**.
1. Enter the path for the ACL policy.
1. Select the desired permissions for that path.
1. Click **Add rule** to define capabilities for another path.
1. Click **Create policy** to save.

![create acl policy with visual policy editor](/img/gui/visual-policy-editor-create-acl.png)
