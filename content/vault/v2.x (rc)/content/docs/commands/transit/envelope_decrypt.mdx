# transit envelope decrypt

The `transit envelope decrypt` command uses Vault Transit (Enterprise) to 
decrypt data encryption keys stored with the ciphertet to to decrypt
files and streams of arbitrary size without transiting the ciphertext to
Vault.  Decryption and happens on the caller's side.

The caller must have write permissions to the transit mount's `decrypt`
endpoint for the given key encrypting key.

## Examples

Decrypt a multi-terabyte file using `envelope decrypt`:

```
$ vault transit envelope decrypt -o really-big-file2.zip transit/keys/test-key really-big-file.zip.vee
```

## Usage

This command respects core Vault CLI command options.

### Decrypt 

 See `vault envelope decrypt -help` for more information.

 1. `PATH`, the path to the transit key to use for DEK/EDK generation in the format of
    `<mount>/keys/<key-name>`, where `<mount>` is the path to the mount
    (using `-namespace=<ns>` to specify any namespaces), and `<key-name>`
    is the desired name of the key.
    By default, decrypted files will be output to the input file name minus
    `.vee` (**V**ault **E**nvelope **E**ncrypted)
 2. One or more filenames to decrypt OR `-` to decrypt input from stdin.

### Command Options

- `--quiet` (bool: false) - Display no output unless there is an error.
- `--output` `(string: "")` - Output to the specified file.  Can only be used if there 
  is a single input.
- `--suffix` `(string: ".vee") - Suffix to remove from output file names.

AAD must match that given during encrypt or decryption will fail.
