To verify secret recovery, read the secret from the expected path. For example,
if you recovered a secret from a key/value version 1 plugin mounted at the
default path:

```shell-session
$ vault kv get kv/telemetry-systems/api-key

=== Data ===
Key    Value
---    -----
key    9C016BB3-F574-4A24-B8A7-8CE1F1CE2128
```

For a key/value version 2 plugin, the output includes a metadata block showing
the version. The recovered data appears as the latest version:

```shell-session
$ vault kv get kv/telemetry-systems/api-key

========== Secret Path ==========
kv/data/telemetry-systems/api-key

======= Metadata =======
Key                Value
---                -----
created_time       2018-03-22T02:24:06.945319214Z
custom_metadata    <nil>
deletion_time      n/a
destroyed          false
version            2

==== Data ====
Key    Value
---    -----
key    9C016BB3-F574-4A24-B8A7-8CE1F1CE2128
```
