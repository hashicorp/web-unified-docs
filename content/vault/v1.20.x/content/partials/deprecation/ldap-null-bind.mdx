## LDAP Authentication Method - deny_null_bind parameter ((#ldap-null-bind))

| Announced | Expected end of support | Expected removal |
| :-------: | :---------------------: | :--------------: |
| NOV 2025  | NOV 2025                | NOV 2025         |

The deny_null_bind parameter in the LDAP authentication method is deprecated.

Vault now always denies login attempts with an empty password, regardless of
 the `deny_null_bind` parameter's value:
- If your configuration currently sets `deny_null_bind` = true, no action is required.
- If your configuration currently sets `deny_null_bind` = false, login attempts 
that previously succeeded with an empty password will now fail.

The parameter will be fully removed in the next major release.
