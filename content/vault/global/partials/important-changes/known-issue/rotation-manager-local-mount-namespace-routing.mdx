#### Issue

On Performance Primary Vault clusters, the Rotation Manager incorrectly routes rotation entries for local mounts under a namespace to shared rotation manager storage instead of local storage.

#### Impact

Rotation entries for local mounts under a namespace are stored in shared storage. Upon Vault restarts, the Rotation Manager loses track of these entries because they are in the incorrect storage location.

#### Recommendation

1. Before upgrading to the fixed version, set `disable_automated_rotation=true` on your root config / static role. This will clean up the rotation entry in shared storage. For example, when using the LDAP Secrets Engine:
```shell-session
  $ vault write ldap/config \
      disable_automated_rotation=true
```
1. After upgrading, you can set the parameter to `false` again so that the Rotation Manager can appropriately re-register and store entries in local storage going forward:
```shell-session
  $ vault write ldap/config \
      disable_automated_rotation=false
```
