#### Issue

On performance primary Vault clusters, the rotation manager incorrectly routes rotation entries for local mounts created under a namespace to the shared rotation manager storage instead of local storage.

As a result, the rotation manager can lose track of these entries when Vault restarts because the rotation entries are stored in the wrong location.

#### Recommendation

1. Before upgrading to a fixed version, set `disable_automated_rotation=true` on your root or static role config to clean up the rotation entry in shared storage. For example, to clean entries for the LDAP Secrets plugin:

  ```shell-session
    $ vault write ldap/config \
        disable_automated_rotation=true
  ```

1. After upgrading, you can set `disable_automated_rotation` back to `false` to have the rotation manager re-register and store entries in local storage going forward:

  ```shell-session
    $ vault write ldap/config \
        disable_automated_rotation=false
  ```
