#### Issue

Previously, Vault ran containers as root. Now, Vault checks explicit capability
permissions when the Docker container starts. As a result,  Docker containers
may fail to start with the following error:

```text
unable to set CAP_SETFCAP effective capability: Operation not permitted
```

#### Workaround

While undesirable from a security perspective, you can continue to run the
container as root or specify the environment variable `SKIP_SETCAP=1`
to bypass the capability check.
