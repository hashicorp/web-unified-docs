### Docker image startup failures ((#docker-image-startup-setfcap-failure))

#### Affected Versions
- 1.19.16
- 1.20.10
- 1.21.5
- 2.0.0

#### Issue
Docker containers fail to start with the error:
```
unable to set CAP_SETFCAP effective capability: Operation not permitted
```

#### Workaround
Run the container as root, or specify the environment variable `SKIP_SETCAP=1` to bypass the capability check.
Note that both of these workarounds are undesirable from a security perspective, though the former was the default
prior to this release.
