#### Issue

Azure is eventually consistent, so internal state propagation may lag after updates. When Vault attempts a rotation before synchronization completes, it may fail to remove the old credential. You must clean up orphaned credentials manually.

For example:

```shell-session
$ vault write -f azure/rotate-role/my-role
```

#### Workaround

Wait a few minutes before each `static-rotate` call.
