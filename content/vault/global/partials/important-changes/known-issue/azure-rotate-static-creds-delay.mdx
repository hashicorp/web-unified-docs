### Azure rotate static credentials cannot run in quick succession

<!-- BEGIN: Vault:=v2.x -->

| Change      | Status   | Vault edition | Affected version | Fixed version
| ----------- | -------- | ------------- | ---------------- | -------------
| Known issue | Closed   | Enterprise    | 2.0.0            | No

<!-- END: Vault:=v2.x -->

#### Issue

A brief delay in state propagation occurs because systems on the Azure side may not immediately synchronize internal data. If Vault attempts a rotation before this synchronization is complete, it may fail to remove the old credential.

For example:

```shell-session
$ vault write -f azure/rotate-role/my-role
```

#### Workaround

Wait a few minutes before each `static-rotate` call.
