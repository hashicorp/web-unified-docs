Vault does not permit wildcards (`+`) and globs (`*`) in the rendered output for
identity templates and returns a "permission denied" response.

For example, the following policy template fails if you set
`region="amer/+/us-west"` in the `education` group's metadata:

```hcl
path "group-kv/data/education/{{identity.groups.names.education.metadata.region}}/*" {
    capabilities = [ "create", "update", "read", "delete", "list" ]
}
```

#### Recommendation

Avoid wildcards and globs in the rendered output for identity templates.
