The Vault docker image requires the `IPC_LOCK` capability to run. In prior
versions of the Vault container we'd set `IPC_LOCK` on the vault binary at
runtime via the entrypoint scripts. As we now run the Vault container as an
unprivileged user we have to set it at build time in order for the binary to
have the capabilities necessary to lock the memory.

#### Recommendation

Add the `IPC_LOCK` capability to the container or pod, or run the container as
root.

The latter is not recommended from a security perspective, but is an option if
you are unable to add the capability to the container or pod.
