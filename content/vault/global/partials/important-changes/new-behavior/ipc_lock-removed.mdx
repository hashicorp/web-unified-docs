We removed the `IPC_LOCK` Linux capability requirement from official Vault
container images while preserving the non-root `vault` user runtime model.

The change better aligns the images with hardened container platforms that
already enforce non-root execution and restrict added capabilities, including
many Kubernetes, OpenShift, and managed container environments.


#### Recommendation

To avoid leaking data, especially if swap is part of the host memory plan, run
Vault on hosts with swap disabled or encrypted, and set `disable_mlock` to
`true`.

Refer to the
[Vault Kubernetes security considerations](/vault/tutorials/kubernetes/kubernetes-security-concerns#ensure-mlock-is-enabled) tutorial
and the
[`disable_mlock` configuration reference](/vault/docs/configuration#disable_mlock)
for more information.
