# Test policies

Once you have authored a policy, write tests that verify that your policy works as intended. Then, run your tests before you implement your new policy and whenever you update your policy.

## Author policy tests

Each test file will include a set of mock providers, resources, data sources, and modules that Terraform policy will use to test your policies. Policy test files must end with `.policytest.hcl`. We recommend that you include your tests in the same VCS repository as your policies. Refer to [Write policies - Directory structure](/terraform/policy/write#directory-structure) for more information about organizing your policies and tests.

To learn how to write policy tests, refer to [Policy test blocks](/terraform/policy/test/test-blocks).

## Run tests

Use the `tfpolicy` CLI to run your tests locally before you enforce them against your infrastructure, and whenever you update your policies.

To learn how to run policy tests, refer to [Run policy tests](/terraform/policy/test/run-tests).

To install the `tfpolicy` CLI, refer to [Install the Terraform policy CLI](/terraform/policy/install).
