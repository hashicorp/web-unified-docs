# Operators

Terraform policy supports standard operators for writing policy conditions. Terraform policy operators behave like their Terraform counterparts.

## Comparison operators

- `==`: Equal to
- `!=`: Not equal to
- `<`: Less than
- `>`: Greater than
- `<=`: Less than or equal to
- `>=`: Greater than or equal to

## Logical Operators

- `&&`: Logical and
- `||`: Logical or
- `!`: Logical not

## Examples

```hcl
resource_policy "aws_instance" "example" {
  enforce {
    condition     = attrs.instance_type == "t3.micro"
    error_message = "Instance type must be t3.micro"
  }
  
  enforce {
    condition     = attrs.monitoring == true && attrs.ebs_optimized == true
    error_message = "Instance must have monitoring and EBS optimization enabled"
  }
}
```
