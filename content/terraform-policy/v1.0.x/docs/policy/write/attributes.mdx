# Attributes and meta-arguments

Within any policy block (`resource_policy`, `provider_policy`, or `module_policy`), you have access to data about your configuration, infrastructure state, and environment through attributes and meta-arguments. Attributes provide data about your configuration and infrastructure, while meta-arguments provide data about your environment from Terraform itself.

## Attributes

Attributes correspond to the provider, resource, or module attributes defined in Terraform. When it evaluates your policies, Terraform policy assigns values to these attributes based on your configuration, plan, or state, depending on when those values are available. For provider attributes, Terraform can compute them by reading your configuration pre-plan, so it does so. For provider and module policies, the values of some attributes will be available at plan time. Other values depend on the successful completion of the apply step, and are marked as `(known after apply)` in Terraform plan output. Terraform will read the value from your state after applying the change.

Attribute values can be complex types like maps and lists, which you can reference in your policies via dot notation (For example: `attrs.tags.environment`).

## Meta-arguments

These are built-in metadata fields provided by the Terraform policy engine. You can use meta-arguments to provide context in your policies, such as where a resource lives in the module tree or what version of a provider is being used.

### The `filter` meta-argument

Terraform policy will apply policies to the providers, resources, or modules which match the first label in the policy's block, filtered by the value of the optional `filter` meta-argument, if provided.

The value of the filter meta attribute must evaluate to a boolean value. Use comparison expressions to compare against attributes and other values. For example:

```hcl
resource_policy "aws_instance" "example" {
  filter = attrs.tags.env == "testing"

# ...
}
```

### Module policy meta-arguments

The following meta-arguments are available in `module_policy` blocks:

| Name | Description | Type | Example |
|------|-------------|------|---------|
| `source` | The full source of the module. | String | `"app.terraform.io/my-org/vpc/aws"`, `"./modules/networking"` |
| `version` | The resolved version of the module. | String | `"2.1.0"`, `"1.0.0"` |
| `address` | Address of the module within Terraform. | String | `"module.vpc"`, `"module.networking.module.subnets"` |
