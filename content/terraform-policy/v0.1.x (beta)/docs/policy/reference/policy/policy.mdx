# `policy`

The `policy` block configures how Terraform policy evaluates your policies, including which version of Terraform to use to perform the evaluation.

@include "partials/beta.mdx"

## Configuration model

```hcl
policy {
  terraform_config {
    required_version = "<version_constraint>"
  }
  plugins {
    <plugin_name> = {
      source = "<path_to_binary>"
    }
  }
}
```

## Complete configuration example

```hcl
policy {
  terraform_config {
    required_version = ">= 1.16.0"
  }
  plugins {
    example = {
      source = "../plugins/bin/example"
    }
    custom_validator = {
      source = "../plugins/bin/custom-validator"
    }
  }
}
```

## Specification

### `terraform_config` block

The `terraform_config` block defines settings which are specific to Terraform.

#### Arguments

| Argument | Required | Type | Description |
|----------|----------|------|-------------|
| `required_version` | No | String | A version constraint that validates the current Terraform CLI version. Uses the same version constraint syntax as Terraform itself. |

The `required_version` attribute uses the same version constraint syntax as Terraform itself. Refer to [Version constraints - Version constraint syntax](/terraform/language/expressions/version-constraints#version-constraint-syntax) for more information.

### `plugins` block

The `plugins` block allows you to refer to binary plugins that provide custom functions you can use in your policies. Each plugin is defined as a named block within the `plugins` block.

#### Arguments

| Argument | Required | Type | Description |
|----------|----------|------|-------------|
| `source` | Yes | String | The relative path to a binary in the same VCS repository as your policies. The binary must be a valid Terraform policy Go plugin server. |

Refer to [Policy plugins](/terraform/policy/write/plugins) for more information about creating and using policy plugins.

In your policies, reference your functions with the syntax `plugin::<plugin_name>::<function_name>(<function_arguments>...)`.

## Examples

### Require minimum Terraform version

```hcl
policy {
  terraform_config {
    required_version = ">= 1.16.0"
  }
}
```

This example requires that the Terraform version used to evaluate these policies be greater than or equal to "1.16.0".

### Configure multiple plugins

```hcl
policy {
  plugins {
    example = {
      source = "../plugins/bin/example"
    }
    validator = {
      source = "../plugins/bin/validator"
    }
  }
}
```

This example configures two plugins that can be used in your policies with the `plugin::example::<function_name>` and `plugin::validator::<function_name>` syntax.

### Complete configuration

```hcl
policy {
  terraform_config {
    required_version = "~> 1.16"
  }
  plugins {
    custom = {
      source = "../plugins/bin/custom"
    }
  }
}
```

This example combines both Terraform version requirements and plugin configuration.
