# The `provider_policy {}` block

Use the `provider_policy {}` block to define policies for Terraform providers. The first label of the block identifies the provider name, such as `aws`, `google`, or `azurerm`, and the second label is the policy name. You can use a wildcard (*) for the provider name to apply a single policy to every provider defined in the configuration.

@include "partials/beta.mdx"

## Configuration model

```hcl
provider_policy "<provider_name>" "<policy_name>" {
  operation         = "<operation>"
  enforcement_level = "<level>"
  filter            = <boolean_expression>

  locals {
    name = <value>
  }

  enforce {
    condition     = <boolean_expression>
    error_message = "<message>"
    info_message  = "<message>"
  }

  enforce { ... }
}
```

## Complete configuration example

```hcl
provider_policy "aws" "standard_configuration" {
  operation         = ["create", "update"]
  enforcement_level = "mandatory"

  locals {
    version_constraint = core::semverconstraint(meta.version, ">= 5.0.0")
  }

  enforce {
    condition     = attrs.region == "us-east-1"
    error_message = "All AWS resources must be deployed in the us-east-1 region."
  }

  enforce {
    condition     = meta.source == "hashicorp/aws"
    error_message = "Only the official HashiCorp AWS provider is permitted."
  }

  enforce {
    condition     = local.version_constraint
    error_message = "The AWS provider version must be at least 5.0.0."
  }
}
```

## Specification

### Labels

| Label | Required | Description |
|-------|----------|-------------|
| First label | Yes | The provider name (For example: `aws`, `google`, `azurerm`). Use `*` to match all providers. |
| Second label | Yes | A unique name for this policy within the policy file. |

### Arguments

Provider policy blocks support the following arguments.

| Argument | Required | Type | Default | Description |
|----------|----------|------|---------|-------------|
| `operation` | No | List of Strings | `["create", "update"]` | The operations during which to enforce this policy. Valid values: `"create"`, `"update"`, `"delete"`. |
| `enforcement_level` | No | String | `"advisory"` | The enforcement level for the policy. Valid values: `"advisory"`, `"mandatory"`, `"mandatory-overridable"`. |
| `filter` | No | Boolean expression | - | Filter which providers the policy applies to based on their attributes. |

### Attributes

Within the `provider_policy {}` block, you can reference the following attirubites.

| Object | Description |
|--------|-------------|
| `attrs` | Provides access to the arguments defined in the provider block, such as region settings or account identifiers. |
| `meta` | Provides access to Terraform-defined information about the provider, including `source`, `version`, and `alias`. |

#### `meta` object attributes

The following meta-arguments are available in `provider_policy` blocks.

| Name | Description | Type | Example |
|------|-------------|------|---------|
| `meta.address` | Address of the provider within Terraform. | String | `"provider["registry.terraform.io/hashicorp/aws"]"` |
| `meta.alias` | Alias given to the provider. | String | `"us-west"`, `"production"` |
| `meta.name` | The local name of the provider. | String | `"aws"`, `"azurerm"` |
| `meta.namespace` | The provider's registry namespace. | String | `"hashicorp"`, `"example-corp"` |
| `meta.source` | The full source of the provider. | String | `"hashicorp/aws"`, `"hashicorp/azurerm"` |
| `meta.type` | The type of provider. | String | `"aws"`, `"azurerm"` |
| `meta.version` | The resolved version of the provider. | String | `"5.0.0"`, `"3.75.0"` |

### The `enforce {}` block

Each `provider_policy {}` block must contain one or more `enforce {}` blocks to define the conditions that must evaluate to `true` for the policy to pass.

#### Arguments

Enforce blocks support the following arguments.

| Argument | Required | Type | Description |
|----------|----------|------|-------------|
| `condition` | Yes | Boolean expression | A boolean expression that must evaluate to `true` for the policy to pass. |
| `error_message` | Yes | String | The error message to display when the condition evaluates to `false`. |
| `info_message` | No | String | An informational message to display regardless of whether the policy passes or fails. |

You can include one or more `enforce {}` blocks in each policy block. If any of the enforce blocks' conditions fail during evaluation, then Terraform policy will fail the policy and return the corresponding error messages.

### The `locals {}` block

Each `provider_validation_policy {}` block may contain `locals {}` blocks that define values that are local to the policy. These local values override `locals {}` blocks defined at the top level of the policy file. Refer to [The `locals {}` block](/terraform/policy/reference/policy/locals) for more information.

## Examples

### Enforce provider source and version

```hcl
provider_policy "aws" "approved_provider" {
  enforcement_level = "mandatory"

  enforce {
    condition     = meta.source == "hashicorp/aws"
    error_message = "Only the official HashiCorp AWS provider is permitted."
  }

  enforce {
    condition     = core::semverconstraint(meta.version, ">= 5.0.0")
    error_message = "The AWS provider version must be at least 5.0.0."
  }
}
```

### Enforce provider configuration

```hcl
provider_policy "aws" "region_check" {
  enforce {
    condition     = attrs.region == "us-east-1"
    error_message = "All AWS resources must be deployed in the us-east-1 region."
  }
}
```

### Use wildcard to match all providers

```hcl
provider_policy "*" "require_official_providers" {
  enforcement_level = "mandatory"

  enforce {
    condition     = core::contains(["hashicorp", "terraform-providers"], core::split("/", meta.source)[0])
    error_message = "Only official HashiCorp providers are allowed. Provider ${meta.source} is not permitted."
  }
}
```

### Filter by provider alias

```hcl
provider_policy "aws" "secondary_region_check" {
  filter = meta.alias == "secondary"

  enforce {
    condition     = attrs.region == "us-west-2"
    error_message = "The secondary AWS provider must use the us-west-2 region."
  }
}
```
