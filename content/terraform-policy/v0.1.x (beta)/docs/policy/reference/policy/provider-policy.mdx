# `provider_policy`

The `provider_policy` block defines policies for Terraform providers. The first label of the block identifies the provider name, such as `aws`, `google`, or `azurerm`, and the second label is the policy name. You can use a wildcard (*) for the provider name to apply a single policy to every provider defined in the configuration.

## Configuration model

The `provider_policy` block supports the following configuration:

- `provider_policy` &nbsp "[`<provider_name>`](#provider-name)" &nbsp "[`<policy_name>`](#policy-name)" &nbsp block
  - [`enforce`](#enforce) &nbsp block | required
    - [`condition`](#enforce-condition) &nbsp boolean expression | required
    - [`error_message`](#enforce-error-message) &nbsp string | optional
    - [`info_message`](#enforce-info-message) &nbsp string | optional
  - [`enforcement_level`](#enforcement-level) &nbsp string | optional
  - [`filter`](#filter) &nbsp boolean expression | optional
  - [`locals`](#locals) &nbsp block | optional
    - `<name>` &nbsp any

## Complete configuration example

The following example demonstrates a provider_policy block with all configuration options specified:

<CodeBlockConfig filename="policies/example.policy.hcl">

```hcl
provider_policy "aws" "standard_configuration" {
  enforcement_level = "mandatory"
  filter            = meta.alias == null

  locals {
    version_constraint = core::semverconstraint(meta.version, ">= 5.0.0")
  }

  enforce {
    condition     = attrs.region == "us-east-1"
    error_message = "All AWS resources must be deployed in the us-east-1 region."
    info_message  = "Checking provider region: ${attrs.region}"
  }

  enforce {
    condition     = meta.source == "hashicorp/aws"
    error_message = "Only the official HashiCorp AWS provider is permitted."
  }

  enforce {
    condition     = local.version_constraint
    error_message = "The AWS provider version must be at least 5.0.0."
  }
}
```

</CodeBlockConfig>

## Specification

The `provider_policy` block supports the following configuration.

### `provider_policy "<TYPE>" "<LABEL>"`

You must set the following arguments in the `provider_policy` block:

- `TYPE`: Specifies the provider name, such as `aws`, `google`, `azurerm`. You can use the `*` wildcard to match all providers. 
- `LABEL`: Specifies a unique name for this policy within the policy file. 

### `enforcement_level`

Specifies the enforcement level for the policy. You can specify one of the following values:

- `"advisory"`: Allows the operation to continue but prints a warning when if the conditions of the policy aren't met. 
- `"mandatory"`: Stops the operation if the conditions of the policy aren't met.
- `"mandatory_overridable"`: Stops the operation if the conditions of the policy aren't met, but allows a user with appropriate permissions to override the policy and continue. 

  - Data type: String
  - Default: `"mandatory"`

### `filter` 

Specifies a Boolean expression that filters which provider configurations the policy applies to based on their attributes. The left side of the expression references an attribute in the provider configuration. The right side specifies an attribute value. When the expression evaluates to `true`, the policy applies to the provider.

You can prefix the left side of the expression with the `meta` object. This object allows you to select a Terraform-specific argument in the provider configuration, such as `source`, `version`, and `alias`. Refer to [`meta` object attributes](#meta-object-attributes) for more information.  

You can also use the `attrs` object in your expression to access arguments defined in the provider block, such as region settings or account identifies. The attributes you can select with the `attrs` depend on the provider. 

- Data type: Boolean expression
- Default: None

#### `meta` object attributes

The following meta-arguments are available in `provider_policy` blocks.

| Name | Description | Type | Example |
|------|-------------|------|---------|
| `meta.alias` | Alias given to the provider. | String | `"us-west"`, `"production"` |
| `meta.name` | The local name of the provider. | String | `"aws"`, `"azurerm"` |
| `meta.namespace` | The provider's registry namespace. | String | `"hashicorp"`, `"example-corp"` |
| `meta.source` | The full source of the provider. | String | `"hashicorp/aws"`, `"hashicorp/azurerm"` |
| `meta.type` | The type of provider. | String | `"aws"`, `"azurerm"` |
| `meta.version` | The resolved version of the provider. | String | `"5.0.0"`, `"3.75.0"` |

### The `enforce {}` block

Each `provider_policy {}` block must contain one or more `enforce {}` blocks to define the conditions that must evaluate to `true` for the policy to pass.

#### Arguments

Enforce blocks support the following arguments.

| Argument | Required | Type | Description |
|----------|----------|------|-------------|
| `condition` | Yes | Boolean expression | A boolean expression that must evaluate to `true` for the policy to pass. |
| `error_message` | No | String | The error message to display when the condition evaluates to `false`. |
| `info_message` | No | String | An informational message to display regardless of whether the policy passes or fails. |

You can include one or more `enforce {}` blocks in each policy block. If any of the enforce blocks' conditions fail during evaluation, then Terraform policy will fail the policy and return the corresponding error messages.

### The `locals {}` block

Each `provider_policy {}` block may contain a `locals {}` block that defines values that are local to the policy. These local values override `locals {}` blocks defined at the top level of the policy file. Refer to [The `locals {}` block](/terraform/policy/reference/policy/locals) for more information.

## Examples
The following examples show how to write configuration for common use cases.

### Enforce provider source and version

The following policy includes a condition that sets the minimum version of the provider to 5.0.0. The condition accesses provider version using the `meta.version` selector and passes it to the `core::semverconstraint` function.

```hcl
provider_policy "aws" "approved_provider" {
  enforcement_level = "mandatory"

  enforce {
    condition     = meta.source == "hashicorp/aws"
    error_message = "Only the official HashiCorp AWS provider is permitted."
  }

  enforce {
    condition     = core::semverconstraint(meta.version, ">= 5.0.0")
    error_message = "The AWS provider version must be at least 5.0.0."
  }
}
```

### Enforce provider configuration

```hcl
provider_policy "aws" "region_check" {
  enforce {
    condition     = attrs.region == "us-east-1"
    error_message = "All AWS resources must be deployed in the us-east-1 region."
  }
}
```

### Use wildcard to match all providers

```hcl
provider_policy "*" "require_official_providers" {
  enforcement_level = "mandatory"

  enforce {
    condition     = core::contains(["hashicorp", "terraform-providers"], core::split("/", meta.source)[0])
    error_message = "Only official HashiCorp providers are allowed. Provider ${meta.source} is not permitted."
  }
}
```

### Filter by provider alias

```hcl
provider_policy "aws" "secondary_region_check" {
  filter = meta.alias == "secondary"

  enforce {
    condition     = attrs.region == "us-west-2"
    error_message = "The secondary AWS provider must use the us-west-2 region."
  }
}
```
