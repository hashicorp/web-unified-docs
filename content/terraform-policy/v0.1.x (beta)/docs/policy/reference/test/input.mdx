# `inputs`

You can set policy input values for an entire test file or for an individual test case with the `inputs {}` block. You can use the block at the top level or nest it in a provider, resource, or module block. Values set on a test case override values set at the top level.

Policy and test files use different block names. Declare an input in a `.policy.hcl` file with a singular, labeled `input "<name>" {}` block. Assign values to those declarations in a `.policytest.hcl` file with an unlabeled, plural `inputs {}` block. The block names are not interchangeable.

## Configuration model

```hcl
# Top-level inputs for entire test file
inputs {
  <input_name> = <input_value>
}

# Block-level inputs for specific test case
<block_type> "<label>" "<name>" {
  inputs {
    <input_name> = <input_value>
  }
}
```

## Complete configuration example

```hcl
# Top-level inputs apply to all test cases
inputs {
  require_encryption = true
  allowed_regions    = ["us-east-1", "us-west-2"]
}

resource "aws_ebs_volume" "encrypted" {
  attrs = {
    availability_zone = "us-east-1a"
    size              = 10
    encrypted         = true
  }
}

resource "aws_ebs_volume" "unencrypted_allowed" {
  # Override top-level input for this specific test case
  inputs {
    require_encryption = false
  }
  
  attrs = {
    availability_zone = "us-east-1a"
    size              = 10
    encrypted         = false
  }
}
```

## Specification

### Syntax

The `inputs {}` block contains one or more assignments. Each assignment sets a value for an `input` block declared in the policy files under test. Test input values are not available as expression variables in test-file `attrs`, `meta`, or `locals` values.

### Precedence

When Terraform policy runs your tests, it will set values for inputs in your policies through the following methods, in order of precedence:

1. In an `inputs {}` block inside a provider, resource, or module block (highest precedence)
2. In an `inputs {}` block at the top level of each test file
3. On the command line with `--input` option
4. From a `TFPOLICY_INPUT_name` environment variable matching the name of the input
5. From a variable definition file, with the `--input-file` option
6. From an environment variable with a name exactly matching the name of the input
7. The default value of the input configured in the policy file (lowest precedence)

## Examples

### Top-level inputs for all test cases

```hcl
inputs {
  environment        = "production"
  require_encryption = true
}

resource "aws_ebs_volume" "prod_encrypted" {
  attrs = {
    availability_zone = "us-east-1a"
    size              = 10
    encrypted         = true
    tags = {
      Environment = "production"
    }
  }
}

resource "aws_instance" "prod_instance" {
  attrs = {
    instance_type = "t3.micro"
    ami           = "ami-12345678"
    tags = {
      Environment = "production"
    }
  }
}
```

### Override inputs per test case

```hcl
inputs {
  max_volume_size = 100
}

resource "aws_ebs_volume" "within_limit" {
  attrs = {
    availability_zone = "us-east-1a"
    size              = 50
  }
}

resource "aws_ebs_volume" "custom_limit" {
  inputs {
    max_volume_size = 200
  }
  
  attrs = {
    availability_zone = "us-east-1a"
    size              = 150
  }
}
```

### Complex input values

```hcl
inputs {
  tagging_requirements = {
    required_tags = ["owner", "project", "environment"]
    environment   = "production"
  }
  
  allowed_instance_types = ["t3.micro", "t3.small", "t3.medium"]
}

resource "aws_instance" "compliant" {
  attrs = {
    instance_type = "t3.micro"
    ami           = "ami-12345678"
    tags = {
      owner       = "platform-team"
      project     = "web-app"
      environment = "production"
    }
  }
}
```

### Provider-specific inputs

```hcl
inputs {
  aws_region = "us-east-1"
}

provider "aws" "correct_region" {
  inputs {
    aws_region = "us-east-1"
  }
  
  meta = {
    source  = "hashicorp/aws"
    version = "5.0.0"
  }
  
  attrs = {
    region = "us-east-1"
  }
}

provider "aws" "different_region" {
  inputs {
    aws_region = "us-west-2"
  }
  
  meta = {
    source  = "hashicorp/aws"
    version = "5.0.0"
  }
  
  attrs = {
    region = "us-west-2"
  }
}
```

### Module-specific inputs

```hcl
inputs {
  min_module_version = "2.0.0"
}

module "app.terraform.io/my-org/vpc/aws" "current" {
  meta = {
    source  = "app.terraform.io/my-org/vpc/aws"
    version = "2.0.0"
  }
  
  attrs = {
    name = "vpc"
    cidr = "10.0.0.0/16"
  }
}

module "app.terraform.io/my-org/vpc/aws" "newer" {
  inputs {
    min_module_version = "3.0.0"
  }
  
  meta = {
    source  = "app.terraform.io/my-org/vpc/aws"
    version = "3.0.0"
  }
  
  attrs = {
    name = "vpc"
    cidr = "10.0.0.0/16"
  }
}
```

### Testing with different input combinations

```hcl
inputs {
  encryption_enabled = true
  kms_key_required   = false
}

resource "aws_ebs_volume" "encrypted_no_kms" {
  attrs = {
    availability_zone = "us-east-1a"
    size              = 10
    encrypted         = true
  }
}

resource "aws_ebs_volume" "encrypted_with_kms" {
  inputs {
    kms_key_required = true
  }
  
  attrs = {
    availability_zone = "us-east-1a"
    size              = 10
    encrypted         = true
    kms_key_id        = "arn:aws:kms:us-east-1:123456789012:key/12345678"
  }
}

resource "aws_ebs_volume" "unencrypted" {
  inputs {
    encryption_enabled = false
  }
  
  expect_failure = true
  
  attrs = {
    availability_zone = "us-east-1a"
    size              = 10
    encrypted         = false
  }
}
```
