# Write and run policy tests

Once you have authored a policy, write tests that verify that your policy works as intended. Then, run your tests before you implement your new policy and whenever you update your policy.

@include "partials/beta.mdx"

## Author policy tests

Each test file will include a set of mock providers, resources, data sources, and modules that Terraform policy will use to test your policies. Policy test files must end with `.policytest.hcl`. We recommend that you include your tests in the same VCS repository as your policies. Refer to [Write policies - Directory structure](/terraform/policy/write#directory-structure) for more information about organizing your policies and tests.

To learn how to write policy tests, refer to the [Policy test reference documentation](/terraform/policy/reference/test).

## Run tests

Use the `tfpolicy test` command to run your tests locally before you enforce them against your infrastructure, and whenever you update your policies.

### Overview

```shell-session
$ tfpolicy test --policies=/path/to/policies
```

You can provide separate paths to the policy files and the test files. If you only specify the path to your policy files, the tests in that path will be executed.

```shell-session
$ tfpolicy test --policies=/path/to/policies --tests=/path/to/tests
```

When it runs your tests, Terraform policy will load values for input variables from your local environment, with the prefix `TF_VAR_`. For example:

```shell-session
$ export TF_VAR_instance_type=t3.medium && tfpolicy test --policies=/path/to/policies --tests=/path/to/tests
```

Refer to [`tfpolicy test` reference](/terraform/policy/reference/cli/test) for more information about the `tfpolicy test` command.
