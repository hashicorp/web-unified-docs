# Write policies

Write and test your Terraform policies on your local machine, and then commit them to version control to use them with HCP Terraform.

@include "partials/beta.mdx"

## Policy files

Terraform policy evaluates policies in files with the `.policy.hcl` file extension. You can create policy files in any directory, passing your policy file path to `tfpolicy` when validating or testing policies. Unlike Terraform, Terraform policy evaluates each policy file in a given directory separately.

### Version control

In order for HCP Terraform to evaluate your policies when provisioning infrastructure, you must host your policy files in a Version Control System (VCS) repository that you will connect to HCP Terraform. Refer to [VCS Setup for Terraform policy](/terraform/cloud-docs/workspaces/policy-enforcement/terraform-policy-vcs) for more information.

### Directory structure

We recommend that you group sets of policies that you intend to enforce together into a VCS repository and create a policy set to enforce them in HCP Terraform. In addition to the policy files, we recommend that you include tests for your policies in the same VCS repository. If you use policy plugins to implement custom functions for your policies to use, you must also commit the plugin binary to your VCS repository.

### File naming conventions

- **Policy files** must use `.policy.hcl` extension
- **Policy test files** must use `.policytest.hcl` extension
- Use descriptive names that indicate the policy's purpose
- Keep related policies and tests in the same VCS repository

## Policy syntax

Terraform policy's configuration language is based on HCL. You will write configuration blocks to configure Terraform policy, define input variables and locals, and write policies for Terraform providers, resources, and modules.

### Configure Terraform policy

Use the `policy {}` block to configure Terraform policy itself. Refer to [Policy blocks - The `policy {}` block](/terraform/policy/write/policy-blocks#the-policy--block) for more information.

### Policy blocks

Terraform policy's configuration language defines blocks to allow you to write policies for Terraform providers, resources, and modules. Refer to the following for more information:

- To write policies for Terraform providers, refer to [Policy blocks - The `provider_policy {}` block](/terraform/policy/write/policy-blocks#the-provider_policy--block).
- To write policies for Terraform resources, refer to [Policy blocks - The `resource_policy {}` block](/terraform/policy/write/policy-blocks#the-resource_policy--block).
- To write policies for Terraform modules, refer to [Policy blocks - The `module_policy {}` block](/terraform/policy/write/policy-blocks#the-module_policy--block).

### Attributes

Like Terraform configuration, policy blocks contain attributes that allow you to define which Terraform the attributes your policy will validate and meta-arguments to configure the policy itself. Refer to [Attributes and meta-arguments](/terraform/policy/write/attributes) for more information.

### Operators

Terraform policy supports most (FIXME: All?) Terraform operators. Refer to [Operators and conditionals](/terraform/policy/write/operators) for more information.

### Functions

Terraform policy includes built-in functions to fetch resources and data sources, as well as most Terraform library functions. Refer to [Policy functions](/terraform/policy/write/functions) for more information.

#### Plugins

You can write custom Terraform policy functions as plugins in Golang. Refer to [Policy plugins](/terraform/policy/write/plugins) for more information.

### Input variables and locals

Terraform policy supports input variables and local values to allow you to parameterize your policy configuration. Refer to [Inputs and locals](/terraform/policy/write/inputs-and-locals) for more information.
