# Built-in functions

Terraform policy includes built-in functions you can use when you write policies. You can also use many built-in Terraform functions, and write function plugins to implement reusable logic such as HTTP functions or arithmetic operations.

## Terraform policy functions

Terraform policy includes built-in functions to allow you to retrieve resources and data sources dynamically within your policies, make HTTP requests, and compare semantic version constraints:

- [`core::getresources`](#the-getresources-function)`(string, map)`: Return a list of resources that match the specified type and filter.
- [`core::getdatasource`](#the-getdatasource-function)`(string, map)`: Return a data source that matches the specified type and filter.
- [`core::gethttprequest`](#the-gethttprequest-function)`(string, map)`: Make an HTTP GET request and return the results.
- [`core::semverconstraint`](#the-semverconstraint-function)`(string, string)`: Validate that a given version matches a version constraint.

### The `getresources` function

Use the `getresources` function to return a list of all resources of the specified type that match the filter, if any. You can also filter by attributes whose values are computed by Terraform, even if they are not explicitly defined in your configuration.

```hcl
locals {
  # Retrieve an s3 bucket with a specific name from the configuration.
  buckets_specific = core::getresources("aws_s3_bucket", {
    bucket = "bucketname"
  })
  
  # Retrieve all encrypted s3 buckets.
  buckets_s3 = core::getresources("aws_s3_bucket", {
    encrypted = true
  })
}
```

### The `getdatasource` function

Use the `getdatasource` function to return a data source of the specified type that matches the filter. Terraform policy passes the filtering attributes to the provider, and returns any matching data sources from the provider. Since this function returns a single data source, use filters if needed to specify a single data source.

```hcl
locals {
  # Retrieve a lambda function with a specific name
  function = core::getdatasource("aws_lambda_function", {
    function_name = "example-function"
  })
}
```

### The `gethttprequest` function

The `core::gethttprequest` function makes an GET request to the given URL with optional header content and returns the statusCode, status and body.

### The `semverconstraint` function

The `core::semverconstraint` function compares the version given in the first argument with the version constraint given in the second argument. It returns `true` if the version matches the constraint, and `false` otherwise.

```hcl
module_policy "app.terraform.io/my-org/vpc/aws" "version_check" {
  enforce {
    condition     = core::semverconstraint(meta.version, ">= 2.0.0")
    error_message = "The VPC module (${meta.address}) must be version 2.0.0 or higher."
  }
}
```

## Terraform library functions

Terraform policy also supports a number of standard Terraform library functions, which are in the 'core' namespace, referenced in policy configuration with `core::<function-name>`.

### Numeric functions

- [`core::abs`](/terraform/language/functions/abs)`(number)`: Returns the absolute (positive) value of a given number.
- [`core::ceil`](/terraform/language/functions/ceil)`(number)`: Rounds a number up to the nearest whole number.
- [`core::floor`](/terraform/language/functions/floor)`(number)`: Rounds a number down to the nearest whole number.
- [`core::log`](/terraform/language/functions/log)`(base, number)`: Returns the logarithm of a number to a given base.
- [`core::max`](/terraform/language/functions/max)`(number, number, ...)`: Returns the largest of a set of numbers.
- [`core::min`](/terraform/language/functions/min)`(number, number, ...)`: Returns the smallest of a set of numbers.
- [`core::parseint`](/terraform/language/functions/parseint)`(string, base)`: Parses a string as a whole number in the given base.
- [`core::pow`](/terraform/language/functions/pow)`(number, power)`: Returns the result of a number raised to a given power.
- [`core::signum`](/terraform/language/functions/signum)`(number)`: Returns a number indicating the sign of the input: -1 for negative, 0 for zero, and 1 for positive.

### String functions

- [`core::chomp`](/terraform/language/functions/chomp)`(string)`: Removes any newline characters from the end of a string.
- [`core::format`](/terraform/language/functions/format)`(spec, values...)`: Produces a string by formatting other values according to a specification string.
- [`core::indent`](/terraform/language/functions/indent)`(num_spaces, string)`: Adds a specified number of spaces to the beginning of each line in a multi-line string.
- [`core::join`](/terraform/language/functions/join)`(separator, list)`: Concatenates all elements of a list of strings into a single string.
- [`core::lower`](/terraform/language/functions/lower)`(string)`: Converts all uppercase letters in a string to lowercase.
- [`core::regex`](/terraform/language/functions/regex)`(pattern, string)`: Returns the first substring that matches the regular pattern.
- [`core::regexall`](/terraform/language/functions/regexall)`(pattern, string)`: Returns a list of all substrings that match the regular pattern.
- [`core::split`](/terraform/language/functions/split)`(separator, string)`: Splits a string into a list of substrings.
- [`core::strrev`](/terraform/language/functions/strrev)`(string)`: Reverses the characters in a string.
- [`core::substr`](/terraform/language/functions/substr)`(string, offset, length)`: Returns a substring starting at offset with the specified length.
- [`core::title`](/terraform/language/functions/title)`(string)`: Converts the first letter of each word to uppercase.
- [`core::trim`](/terraform/language/functions/trim)`(string, cutset)`: Removes leading or trailing characters specified in cutset.
- [`core::trimprefix`](/terraform/language/functions/trimprefix)`(string, prefix)`: Removes the specified prefix from the beginning.
- [`core::trimspace`](/terraform/language/functions/trimspace)`(string)`: Removes all leading and trailing whitespace.
- [`core::trimsuffix`](/terraform/language/functions/trimsuffix)`(string, suffix)`: Removes the specified suffix from the end.
- [`core::upper`](/terraform/language/functions/upper)`(string)`: Converts all lowercase letters to uppercase.

### Collection functions

- [`core::can`](/terraform/language/functions/can)`(expression)`: Evaluates an expression and returns true if it produces a result without errors.
- [`core::chunklist`](/terraform/language/functions/chunklist)`(list, chunk_size)`: Splits a single list into fixed-size chunks.
- [`core::coalescelist`](/terraform/language/functions/coalescelist)`(list, list, ...)`: Returns the first list that is not null or empty.
- [`core::compact`](/terraform/language/functions/compact)`(list)`: Returns a new list with any null or empty string elements removed.
- [`core::concat`](/terraform/language/functions/concat)`(list, list, ...)`: Combines two or more lists into a single list.
- [`core::contains`](/terraform/language/functions/contains)`(list, value)`: Determines whether a list contains at least one element equal to the given value.
- [`core::distinct`](/terraform/language/functions/distinct)`(list)`: Returns a new list containing only the unique elements.
- [`core::element`](/terraform/language/functions/element)`(list, index)`: Retrieves a single element from a list by its index.
- [`core::flatten`](/terraform/language/functions/flatten)`(list)`: Collapses a multi-dimensional list into a single-dimensional list.
- [`core::formatlist`](/terraform/language/functions/formatlist)`(spec, values...)`: Produces a list of strings by formatting values.
- [`core::keys`](/terraform/language/functions/keys)`(map)`: Returns a list of all the keys in a given map.
- [`core::length`](/terraform/language/functions/length)`(collection)`: Returns the number of elements in a list, map, or string.
- [`core::merge`](/terraform/language/functions/merge)`(map, map, ...)`: Combines multiple maps into a single map.
- [`core::range`](/terraform/language/functions/range)`(limit)` / `core::range(lower, upper)` / `core::range(lower, upper, step)`: Generates a list of numbers.
- [`core::reverse`](/terraform/language/functions/reverse)`(list)`: Reverses the order of elements in a list.
- [`core::setintersection`](/terraform/language/functions/setintersection)`(set1, set2, ...)`: Returns elements common to all given sets.
- [`core::setproduct`](/terraform/language/functions/setproduct)`(set1, set2, ...)`: Returns the Cartesian product of the given sets.
- [`core::setsubtract`](/terraform/language/functions/setsubtract)`(set1, set2)`: Returns elements from set1 not present in set2.
- [`core::setunion`](/terraform/language/functions/setunion)`(set1, set2, ...)`: Returns all unique elements from all given sets.
- [`core::slice`](/terraform/language/functions/slice)`(list, start, end)`: Returns a subset of elements from the original list.
- [`core::sort`](/terraform/language/functions/sort)`(list)`: Sorts the elements of a list in ascending order.
- [`core::values`](/terraform/language/functions/values)`(map)`: Returns a list of all the values in a given map.
- [`core::zipmap`](/terraform/language/functions/zipmap)`(keys, values)`: Creates a map from two lists of equal length.

### Encoding/decoding functions

- [`core::csvdecode`](/terraform/language/functions/csvdecode)`(string)`: Decodes CSV-formatted data into a list of maps.
- [`core::jsondecode`](/terraform/language/functions/jsondecode)`(string)`: Decodes a JSON-formatted string into a Terraform value.
- [`core::jsonencode`](/terraform/language/functions/jsonencode)`(value)`: Encodes a Terraform value into a JSON-formatted string.
- [`core::yamldecode`](/terraform/language/functions/yamldecode)`(string)`: Decodes a YAML-formatted string into a Terraform value.
- [`core::yamlencode`](/terraform/language/functions/yamlencode)`(value)`: Encodes a Terraform value into a YAML-formatted string.

### Terraform-specific functions

- [`core::timeadd`](/terraform/language/functions/timeadd)`(timestamp, duration)`: Adds a duration string to a timestamp.
- [`core::timecmp`](/terraform/language/functions/timecmp)`(timestamp1, timestamp2)`: Compares two timestamps and returns 0 if equal, -1 if first is earlier, or 1 if first is later.
- [`core::timestamp`](/terraform/language/functions/timestamp)`()`: Returns a string representation of the current date and time.
- [`core::try`](/terraform/language/functions/try)`(expression, fallback_expression, ...)`: Evaluates expressions in order and returns the first one that does not produce an error.

## Plugins

In addition to the built-in functions above, you can write plugins to implement your own Terraform policy functions. Refer to [Terraform Policy - Policy plugins](/terraform/policy/write/plugins) for more information.
