# Run policy tests

Once you have written your policy tests, run them using the `tfpolicy test` command.

@include "partials/beta.mdx"

## Overview

```shell-session
$ tfpolicy test --policies=/path/to/policies
```

You can provide the path to both the policy files and the test files. If you only specify the path to your policy files, the tests in that path will be executed.

```shell-session
$ tfpolicy test --policies=/path/to/policies --tests=/path/to/tests
```

Terraform policy will load values for input variables from you local environment, with the prefix `TF_VAR_`. For example:

```shell-session
$ export TF_VAR_instance_type=t3.medium && tfpolicy test --policies=/path/to/policies --tests=/path/to/tests
```

Not all policies run at every stage of a Terraform run. Use the `--evaluation-stage=<stage>` argument to have Terraform policy run the tests either pre-plan (setup) time, or after the plan has been created.

```shell-session
$ tfpolicy test --policies=/path/to/policies --tests=/path/to/tests --evaluation-stage=setup
```

Refer to [Terraform policy - `tfpolicy test` reference](/terraform/policy/reference/cli/test) for more information about the `tfpolicy test` command.

## Set values for inputs

When Terraform policy runs your tests, it will set values for inputs in your policies through the following methods, in order of precendence:

1. In an [`inputs {} block`](/terraform/policy/test/test-blocks#the-input--block) inside a provider, resource, or module block.
1. In an [`inputs {} block`](/terraform/policy/test/test-blocks#the-input--block) at the top level of each test file.
1. On the command line with [`--input` option](/terraform/policy/reference/cli/test#available-options).
1. From a `TFPOLICY_INPUT_name` environment variable matching the name of the input.
1. From a variable definition file, with the [`--input-file` option](/terraform/policy/reference/cli/test#available-options)
1. From an environment variable with a name exactly matching the name of the input.
1. The default value of the input configured in the policy file.

If Terraform policy cannot assign a value to an input through one of the above methods, it will error.

To learn more about policy inputs, refer to [Inputs variables and local values - The `input {}` block](/terraform/policy/write/inputs-and-locals#the-input--block).

## Test output

When you run tests, tfpolicy provides detailed output showing the status of each test case:

```shell-session
$ tfpolicy test --policies=.
# encrypted_ebs.policytest.hcl... running
# resource.aws_ebs_volume.ebs_vol... running
# resource.aws_ebs_volume.ebs_vol... pass
# resource.aws_ebs_volume.ebs_vol2... running
# resource.aws_ebs_volume.ebs_vol2... pass
# encrypted_ebs.policytest.hcl... pass
```

## Debugging Failed Tests

When a test fails, tfpolicy will provide detailed error messages indicating:

- Which policy failed
- Which test case failed
- The specific condition that was not met
- The error message defined in the policy

```text
# resource.aws_instance.test_case... fail
Error: Policy violation
  Expected: pass
  Actual: fail
  Message: Instance type must be t3.micro
```

Use this information to debug failed tests.
