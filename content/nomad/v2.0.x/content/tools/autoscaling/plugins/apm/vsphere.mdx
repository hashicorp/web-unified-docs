# vSphere APM plugin

Use the `vsphere` application performance management (APM) plugin to query
VMware vCenter performance metrics for Nomad Autoscaler checks.

<EnterpriseAlert>
  This functionality only exists in Nomad Autoscaler Enterprise.
</EnterpriseAlert>

## Requirements

The vSphere APM plugin supports vSphere 7.0 and later.

## Agent configuration options

Configure the plugin in the Nomad Autoscaler agent configuration.

```hcl
apm "vsphere" {
  driver = "vsphere"

  config = {
    vcenter_url          = "https://VCENTER_ADDRESS/sdk"
    username             = "VCENTER_USERNAME"
    password             = "VCENTER_PASSWORD"
    insecure_skip_verify = "false"
    datacenter           = "DATACENTER_NAME"
  }
}
```

- `vcenter_url` `(string: <required>)` - The absolute URL for vCenter,
  including the scheme. Use the vCenter endpoint, such as
  `https://VCENTER_ADDRESS/sdk`.

- `username` `(string: <required>)` - The vCenter username used to authenticate
  the plugin.

- `password` `(string: <required>)` - The password for `username`.

- `insecure_skip_verify` `(string: "false")` - Set to `"true"` to skip
  Transport Layer Security (TLS) certificate verification. Keep the default
  value unless you use a test environment with a certificate that you cannot
  trust.

- `datacenter` `(string: "")` - The vCenter datacenter that contains the target
  cluster or resource pool. When omitted, the plugin uses the default
  datacenter.

## Policy configuration options

Set the check `source` to `vsphere`. The `query` must be a JavaScript Object
Notation (JSON) object. Use `jsonencode` to create the query in HashiCorp
Configuration Language (HCL).

```hcl
check "vcenter_cpu" {
  source = "vsphere"
  query = jsonencode({
    entity_type = "cluster"
    entity_name = "nomad-prod"
    aggregation = "avg"
    metric      = "cpu.usage.average"
  })

  query_window = "5m"

  strategy "target-value" {
    target = 70
  }
}
```

The query supports the following fields:

- `entity_type` `(string: <required>)` - The vSphere entity type. Set to
  `cluster` or `resourcepool`.

- `entity_name` `(string: <required>)` - The name of the cluster or the
  inventory path of the resource pool.

- `aggregation` `(string: <required>)` - The aggregation to apply to the metric
  values. Set to `avg`, `max`, `min`, or `sum`.

- `metric` `(string: <required>)` - The vCenter performance counter name, such
  as `cpu.usage.average`.

<Note>

The vSphere plugin does not support an instant `query_window`. Use a
duration-based window instead.

</Note>
