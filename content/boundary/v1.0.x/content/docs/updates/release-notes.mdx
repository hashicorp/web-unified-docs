# Boundary 1.0.x release notes

@include 'release-notes/intro.mdx'

@include 'release-notes/change-tracker-tip.mdx'

## Boundary 1.0.0

GA date: 2026-06-30

Editions: HCP and Enterprise

- **Boundary Desktop version**: [2.6.1](#boundary-desktop-2-6-1)
- **Client Agent version**: [1.0.0](#client-agent-1-0-0)
- **Boundary installer version**: [1.0.0](#boundary-installer-1-0-0)

### New features

- [RDP session recording](/boundary/docs/session-recording) - You can now configure RDP targets to record sessions.
- [SSH session recording recovery](/boundary/docs/session-recording/recovery) - A new feature automatically attempts to recover SSH session recordings if a worker loses the connection during a recording.
- [Helm chart](/boundary/docs/deploy/helm-chart) - You can now install Boundary controllers and workers in Kubernetes environments using a Helm chart.

### Improvements and behavior changes

- [Project-level aliases](/boundary/docs/targets/configuration) - You can now create aliases at the project level. Project-level aliases require suffixes.
- [Grants UI update](/boundary/docs/rbac/manage-roles) - A new code editor featuring autocomplete and linting support was added to help create grant strings for roles. You can select from templates that display [example grant strings](/boundary/docs/rbac/example-roles).
- New `instance_addresses_only` option  - A new option has been added to the [`host-catalogs create`](/boundary/docs/commands/host-catalogs/create#attribute-options) and [`host-catalogs update`](/boundary/docs/commands/host-catalogs/update#attribute-options) attributes for the AWS plugin. The `instance_addresses_only` option lets you limit the syncing of instance addresses so that Boundary automatically selects a host's primary ENI (Elastic Network Interface).

### Bug fixes and security patches

- Updated worker connection handling to avoid CPU spikes.
- Fixed issue with email and name population in event logging.
- Fixed the visibility of the search Cancel button in dark mode.

### Known issues

- The Admin Console uses secure cookies and therefore requires HTTPS. HTTP access is not supported.
- When you update an org scope using the API, the response does not include the `authorized_collection_actions` field. This is a known issue.

#### RDP session recording

- A patented codec, AVC420, requires a license and is unsupported for RDP session recording at this time. Video content that uses H.264/AVC compression may cause a black box to display briefly during RDP session recordings. This is a known issue.
- Microsoft Terminal Services Client (MSTSC) uses a proprietary codec that is not supported for session recording at this time. Video content that uses H.264/AVC compression may not render properly. This is a known issue. There is no workaround, but alternate applications such as FreeRDP might work.
- Session recording fails with the error "Missing credential information" for RDP targets that use a Vault LDAP credential library. This is a known issue that is resolved in [version 1.0.1](#boundary-1-0-1).

## Boundary 1.0.1

GA date: 2026-07-09

Editions: HCP and Enterprise

### Improvements and behavior changes

- The AWS plugin now uses S3 multipart uploads for large objects. Previously, recorded session files could not be over 5 GB in size. This enhancement removes the 5 GB limitation. Refer to session recording [storage considerations](/boundary/docs/session-recording#storage-considerations) for more information.

### Bug fixes and security patches

- Previously, session recording failed with the error "Missing credential information" for RDP targets that used a Vault LDAP credential library. This issue is resolved in version 1.0.1. [Upgrade to the latest version of Boundary](/boundary/tutorials/self-managed-deployment/upgrade-version).
