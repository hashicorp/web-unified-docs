You must have write permissions for a scope to create, edit, or delete a suffix in that scope.
Any user who has the `*` all action grant for a scope can set or remove aliases.

If you want to configure grants to specifically enable adding or removing suffixes for a scope, you can use the following strings.

For a specified org:

```shell-session
$ boundary roles create -scope-id global -name "org-alias-suffix-manager" \

boundary roles add-grants -id r_xyz789 \
  -grant "ids=o_org1;type=scope;actions=set-alias-suffix,remove-alias-suffix"

boundary roles set-grant-scopes -id r_xyz789 \
  -grant-scope-id "this"
```

For a specified org and all of its projects:

```shell-session
$ boundary roles create -scope-id o_org1 -name "project-alias-suffix-manager" \

boundary roles add-grants -id r_xyz789 \
  -grant "ids=*;type=scope;actions=set-alias-suffix,remove-alias-suffix"

boundary roles set-grant-scopes -id r_xyz789 \
  -grant-scope-id "this"
```

For a specified project:

```shell-session
$ boundary roles create -scope-id o_org1 -name "project-alias-suffix-manager" \

boundary roles add-grants -id r_xyz789 \
  -grant "ids=*;type=scope;actions=set-alias-suffix,remove-alias-suffix"

boundary roles set-grant-scopes -id r_xyz789 \
  -grant-scope-id "this"
```
