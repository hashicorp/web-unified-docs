The Project Admin manages a specific project, with full control over host catalogs, targets, credential stores, and sessions in that project.
This role is responsible for the operational aspects of a project's resources.
Use cases could include:
- Setting up target infrastructure
- Configuring access to systems
- Managing credential access
- Managing project-level user access

Recommended permissions for the Project scope:
```shell-session
    ids=<project-id>;type=scope;actions=*
    ids=*;type=host-catalog;actions=*
    ids=*;type=target;actions=*
    ids=*;type=credential-store;actions=*
    ids=*;type=session;actions=*
```
