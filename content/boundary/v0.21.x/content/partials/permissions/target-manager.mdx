The Target Manager creates and manages targets, which are services that users can connect to.
Their responsibilities can include configuring which hosts, credentials, and resources are available to users.
Target Managers are frequently DevOps engineers or system administrators.
Uses cases could include:
- Adding new services to Boundary
- Updating target configurations

Recommended permissions for the Org and/or Project scope:
```shell-session
    type=target;actions=create,list
    ids=*;type=target;actions=read,update,delete,add-host-sources,set-host-sources,remove-host-sources,add-credential-sources,set-credential-sources,remove-credential-sources
```
