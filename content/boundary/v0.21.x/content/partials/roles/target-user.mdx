The Target User can connect to targets by creating sessions.
They have the ability to view and cancel their own sessions.
This role is a basic-end user role intended for accessing protected resources.
Target Users are frequently developers, database administrators, end users requiring server access, or support staff.
Use cases include:
- Connecting to targets
- Viewing session information
- Canceling sessions

We recommend the following permissions for Target Users, assigned at the org or project scope level:
```shell-session
ids=*;type=target;actions=authorize-session,read,list
ids=*;type=session;actions=read:self,cancel:self
```
