The Read-Only User can view all resources, but cannot make any changes.
This role is useful for audit, reporting, and monitoring purposes where visibility is important, but modifications are not required.
Read-Only Users are frequently security engineers, compliance observers, or external auditors.

Use cases include:
- Monitoring for security issues
- Training
- Observing for compliance
- Auditing

We recommend the following permissions for Read-Only Users, assigned at the global, org, or project scope level.

Read and list any resources. View the output of a list command without having any other assigned permissions:
```shell-session
ids=*;type=*;actions=read,list,no-op
```
