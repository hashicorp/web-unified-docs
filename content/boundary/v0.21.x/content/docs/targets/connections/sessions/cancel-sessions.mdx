# Cancel sessions

Sessions are resources that Boundary creates when you connect to a target.

Boundary automatically cancels a session when the session reaches the configured time limit and expires, or when any resource associated with the session is deleted or removed from the target.
Those associated resources could include the host, host set, host catalog, a credential, credential library, credential store, the target, project, organization, user, user account, or the account's authentication method.

If you create a session and no longer need it, you can manually cancel it before it expires.
You may need to cancel a session for security purposes, if you detect unexpected or unauthorized activity.

<Tabs>
<Tab heading="Admin Console" group="admin">

Complete the following steps to cancel a session:

1. Log in to Boundary.
1. In the navigation menu, select **Orgs**.
1. Select the org that contains the project for which you want to view sessions.
1. Select the project.
1. In the navigation menu, select **Sessions**.
1. Click **Cancel** next to the session you want to cancel.

  The session status changes to `canceling` and then `terminated`.

</Tab>
<Tab heading="CLI" group="cli">

Complete the following steps to cancel a session:

1. Log in to Boundary.
1. Use the following command and provide the ID of the session you want to cancel:

    ```shell-session
    boundary sessions cancel -id <SESSION_ID>
    ```

    Boundary displays the following information.
    The session status changes to `canceling` and then `terminated`.
