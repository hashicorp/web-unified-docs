# Overview of sessions

Sessions are resources that Boundary creates when you connect to a target.
The target defines an endpoint and any associated connection information.
For example, you can link a target to a credential library, specify a host, or set a default port for the connection.
When you initiate a connection to the target, Boundary verifies your permissions and then creates a session in the target's project scope using the configured attributes.

When a session is created, Boundary captures a snapshot of the data used to authorize it and stores this information in the Boundary data warehouse.
Boundary never deletes session data from the data warehouse, even after the session ends.
You can view details for sessions that are active, pending, or in the process of being canceled through the Admin Console or CLI.

Boundary automatically cancels a session when it reaches the configured time limit or when a resource associated with the session is deleted or removed from the target.
You can also manually cancel a session if you no longer need it or if you detect unauthorized access.

In a typical workflow, Boundary authorizes a user when the session is established.
However, you can also pre-authorize sessions to save time, support automated workflows, or prevent your users from accessing connection details.
