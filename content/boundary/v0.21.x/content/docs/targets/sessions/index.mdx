# Overview of sessions

Sessions are resources that Boundary creates when you connect to a target.
The target represents an endpoint and any relevant connection information.
For example, you can associate a target with a credential library, choose a specific host to connect to, or configure a default port for the connection.
When you initiate a connection to the target, Boundary confirms that the user has the appropriate permissions, and then creates the session in the target's project scope using the configured attributes.

When the session is established, Boundary captures a snapshot of the data relevant to authorizing the session and stores it in the Boundary data warehouse.
It never deletes any of the session data that is stored in the data warehouse, even after sessions are terminated.
You can view session information for any sessions that are active, pending, or in the process of being canceled using the Admin Console or CLI.

Boundary automatically terminates a session when the session reaches the configured time limit and expires, or when any resource associated with the session is deleted or removed from the target.
You can also manually cancel sessions.

In the standard workflow, Boundary authorizes a user to connect to a target when the session is established.
You can also pre-authorize sessions, however, to save time, enable automated workflows, or prevent your users from having access to connection information.
