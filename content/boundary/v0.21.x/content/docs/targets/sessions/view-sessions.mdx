# View session information

Sessions are resources that Boundary creates when you connect to a target.
Boundary creates sessions in the project scope that contains the target you connect to.

You can view session information for any sessions that are active, pending, or in the process of being canceled.

<Tabs>
<Tab heading="Admin Console" group="admin">

Complete the following steps to view session information:

1. Log in to Boundary.
1. In the navigation menu, select **Orgs**.
1. Select the org that contains the project for which you want to view sessions.
1. Select the project.
1. In the navigation menu, select **Sessions**.

  The **Sessions** page displays the following information about the sessions in that project:

   - Session ID
   - User who initiated the session
   - Target name
   - Start date
   - Session status

</Tab>
<Tab heading="CLI" group="cli">

You can use the `list` or `read` commands to view session information.
The `list` command lists all sessions for a scope, while the `read` command displays detailed information for a specific session.

## View all sessions in a specific scope

Complete the following steps to view all sessions in a specific scope:

1. Log in to Boundary.
1. Use the following command and provide the project scope's ID:

    ```shell-session
    boundary sessions list -scope-id <SCOPE_ID>
    ```

    Boundary displays the following information for any sessions in the project scope:

    ```shell-session
    $ boundary sessions list -scope-id p_1234567890

    Session information:
        ID:               s_h7vBIhH5SZ
        Status:           active
        Created Time:     2026-04-30T15:18:14-07:00
        Expiration Time:  2026-04-30T23:18:14-07:00
        Updated Time:     2026-04-30T15:18:35-07:00
        User ID:          u_1234567890
        Target ID:        ttcp_1234567890
    ```

## View detailed information about a specific session

Complete the following steps to view detailed information about a specific session:

1. Log in to Boundary.
1. Use the following command and provide the session ID:

    ```shell-session
    boundary sessions read -id <SESSION_ID>
    ```

    Boundary displays the following information for the session:

    ```shell-session
    $ boundary sessions read -id s_h7vBIhH5SZ

    Session information:
    ```
