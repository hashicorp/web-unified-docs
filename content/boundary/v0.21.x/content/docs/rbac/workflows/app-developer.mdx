# App developer permissions

App developers typically require view only or limited access to Boundary.
They use the Boundary Desktop UI or CLI to access targets and databases.

## Responsibilities

App developers may be responsible for some or all of the following tasks.

### Access targets

Focus: Onboarding and local environment configuration to access targets

- **Authentication** -  Sign in to specific org scopes using the Desktop UI or CLI to view the authorized resources.
- **Target selection** - Browse or query the list of available targets within the assigned projects using the Desktop UI or CLI.
- **Connection** - Initiate a connection to a remote target through Boundary to work on the resource.
- **Session management** - Monitor or list active connections and disconnect.
- **Alias** - List aliases that point to resources that use transparent sessions.
