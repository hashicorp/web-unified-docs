# Add a data source - GitLab

Vault Radar allows you to connect to any top-level group on [GitLab
Cloud](#add-gitlab-cloud) and [GitLab Self-Managed](#add-gitlab-self-serve).

## Add GitLab Cloud

### Prerequisites

- The group name
- Personal Access Token (with the right set of scopes)

## Create a connection with GitLab Cloud

1. Select **Settings/Data Sources**.

1. Click on the **GitLab Cloud** button.

1. Enter the group name in the **Group name** text field.

1. Enter the personal access token in the **Personal Access Token** text field.

  ![Configure GitLab](/img/docs/vault-radar/gitlab-pat-scopes.png)

1. Click **Next**.

## Add GitLab self-managed

### Prerequisites

- Domain name
- Group name
- Personal Access Token (with the right set of scopes)

## Create a connection with GitLab self-managed

1. Select **Settings/Data Sources**.

1. Click on the **GitLab Self-Managed** button.

1. Enter the domain name in the **Domain name** text field.

1. Enter the group name in the **Group name** text field.

1. Enter the personal access token in the **Personal Access Token** text field.

  ![Configure GitLab](/img/docs/vault-radar/gitlab-pat-scopes.png)

1. Click **Next**.

## Add repositories to Vault Radar

Once you establish a connection, select which repositories you
want to install by selecting a group or all.

<Note>

Vault Radar has a limit of 5000 repositories per connection. If you have more
then 5000 repositories, Vault Radar will select the 5000 repositories with the
most recent activity.

</Note>
