# Add a data source - Confluence Data Center

Vault Radar allows you to connect to any organization on Confluence Data Center.

## Add Confluence Data Center

### Prerequisites

- Confluence Server or Confluence Data Center 7.9 or later
- Domain name for your instance
- [Personal Access Token](https://confluence.atlassian.com/enterprise/using-personal-access-tokens-1026032365.html)

## Create a connection with Confluence Data Center

<Note>

Use the **Confluence Server** button in the HCP Portal to connect to either Confluence
Server or Confluence Data Center.

</Note>

### Scan with HCP Cloud

1. Click **Settings**, then click **Data Sources**.

1. Select **HCP Cloud Scan** and click **Continue**.

1. Click on the **Confluence Server** button.

1. Enter the domain name in the **Enter your Confluence domain name** text field.

1. Enter the personal access token in the **Enter your Confluence personal access token** text field.

1. Click **Next**.

1. Select either the **All active spaces** or **Select spaces to monitor** radio
   button.

1. Click **Finish**.

### Scan with agent

1. Click **Settings**, then click **Data Sources**.

1. Select **Vault Radar Agent Scan**.

1. Click on the **Confluence Server** button.

1. Enter the domain name in the **Enter your Confluence domain name** text field.

1. Create an environment variable for the personal access token in the agent environment.

   **Example:**

   <CodeBlockConfig hideClipboard>
   
   ```shell-session
   $ export CONFLUENCE_TOKEN=my_pat
   ```

   </CodeBlockConfig>

1. Enter the environment variable in the **Enter your Confluence personal access token** text field with the `env://` prefix.
   
   **Example:**
   
   ```shell-session
   $ env://CONFLUENCE_TOKEN
   ```

1. Click **Next**.

1. Select either the **All active spaces** or **Select spaces to monitor** radio
   button.

1. Click **Finish**.
