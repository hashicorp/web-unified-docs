# Add a data source - Jira Cloud

Vault Radar allows you to connect to any organization on Jira Cloud

## Add Jira Cloud

### Prerequisites

- Organization name
- Token creator email address
- [API token](https://support.atlassian.com/atlassian-account/docs/manage-api-tokens-for-your-atlassian-account/)

## Create a connection with Jira Cloud

### Scan with HCP Cloud

1. Click **Settings**, then click **Data Sources**.

1. Click **Add data source**.

1. Select **HCP Cloud Scan** and click **Continue**.

1. Click the **Jira Cloud** button.

1. Enter the Organization name in the **Enter your Jira organization name** text field.

1. Enter the email address of the Jira user that created the API token in
   the **Enter your Jira email of the Atlassian token creator** text field.

1. Enter the API token in the **Enter your Atlassian API token** text field.

1. Click **Next**.

1. Select either the **All active projects** or **Select projects to monitor** radio
   button.

1. Click **Finish**. Your Jira Cloud projects will now begin scanning.

### Scan with agent

1. Click **Settings**, then click **Data Sources**.

1. Click **Add data source**.

1. Select **Vault Radar Agent Scan**.

1. Click the **Jira Cloud** button.

1. Enter the Organization name in the **Organization Name** text field.

1. Create an environment variable for the API token in the agent environment. This should be in the format `token_creator_email:api_token`.
   
   **Example:**

   <CodeBlockConfig hideClipboard>

   ```shell-session
   $ export JIRA_CLOUD_API_TOKEN=myemail@example.com:mytoken
   ```

   </CodeBlockConfig>

1. Enter the environment variable in the **Environment Path** text field with the `env://` prefix.
   
   Example:
   
   ```plaintext
   env://JIRA_CLOUD_API_TOKEN
   ```

1. Click **Next**.

1. Select either the **All active projects** or **Select projects to monitor** radio
   button.

1. Click **Finish**. Your Jira Cloud spaces will now begin scanning.
