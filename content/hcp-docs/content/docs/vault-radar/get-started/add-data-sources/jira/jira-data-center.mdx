# Add a data source - Jira Data Center

Vault Radar allows you to connect to any organization on Jira Data Center.

## Add Jira Data Center

### Prerequisites

- Jira Server or Jira Data Center 8.14 or later
- Domain name for your instance
- [Personal Access Token](https://confluence.atlassian.com/enterprise/using-personal-access-tokens-1026032365.html)

## Create a connection with Jira Data Center

<Note>

Use the **Jira Server** button in the HCP Portal to connect to either Jira
Server or Jira Data Center.

</Note>

### Scan with HCP Cloud

1. Click **Settings**, then click **Data Sources**.

1. Click **Add data source**.

1. Select **HCP Cloud Scan** and click **Continue**.

1. Click on the **Jira Server** button.

1. Enter the domain name in the **Enter your Jira domain name** text field.

1. Enter the personal access token in the **Enter your Atlassian personal access token** text field.

1. Click **Next**.

1. Select either the **All active projects** or **Select projects to monitor** radio
   button.

1. Click **Finish**. Your Jira Server projects will now begin scanning.

### Scan with agent

1. Click **Settings**, then click **Data Sources**.

1. Click **Add data source**.

1. Select **Vault Radar Agent Scan** and click **Continue**.

1. Click on the **Jira Server** button.

1. Enter the domain name in the **Enter your Jira domain name** text field.

1. Create an environment variable for the personal access token in the agent environment.

   **Example:**

   <CodeBlockConfig hideClipboard>
   
   ```shell-session
   $ export JIRA_PERSONAL_ACCESS_TOKEN=my_pat
   ```

   </CodeBlockConfig>

1. Enter the environment variable in the **Enter environment path to your
   Atlassian Environment path to your personal access token** text field with
   the `env://` prefix.
   
   **Example:**
   
   ```plaintext
   env://JIRA_PERSONAL_ACCESS_TOKEN
   ```

1. Click **Next**.

1. Select either the **All active projects** or **Select projects to monitor** radio
   button.

1. Click **Finish**.
