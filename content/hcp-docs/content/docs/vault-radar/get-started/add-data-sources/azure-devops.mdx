# Add a data source - Azure DevOps

Vault Radar allows you to connect to any organization on Azure DevOps cloud.

## Prerequisites

To establish a connection, you need the following:

- Organization and project name
- User email
- Access token with the right set of scopes
    - Code > Read
    - Entitlements > Read
    - Graph > Read
    - Identity > Read
    - Member Entitlement Management > Read
    - Notifications > Read, Write, Manage
    - Project and Team > Read
    - Service Connections > Read
    - User Profile > Read
    - Work Items > Read, Write


## Create a connection with Azure DevOps

1. Select **Settings/Data Sources**.

1. Click on the **Azure DevOps Cloud** button.

1. Enter the organization name in the **Organization** text field.

1. Enter the project name in the **Project** text field.

1. Enter the user email and the access token value.

  ![Configure Azure DevOps](/img/docs/vault-radar/ui-config-azure-devops.png)

  <Tip title="Personal Access Token">

  Refer to the [Azure DevOps
  documentation](https://learn.microsoft.com/en-us/azure/devops/organizations/accounts/use-personal-access-tokens-to-authenticate?view=azure-devops&tabs=Windows)
  for an instructions to create a personal access token.

  </Tip>

1. Click **Next**.

## Add repositories to Vault Radar

Once you establish a connection, select which repositories you
want to install by selecting a group or all.

<Note>

Vault Radar has a limit of 5000 repositories per connection. If you have more
then 5000 repositories, Vault Radar will select the 5000 repositories with the
most recent activity.

</Note>
