# Terraform Enterprise connection

The Terraform Enterprise connection connects your Terraform Enterprise data to the resource graph.

## Allow network API access to Infragraph

To allow Infragraph [API](/terraform/enterprise/api-docs) access to your Terraform Enterprise instance, you must provide an internet-facing API address that is accessible to Infragraph. Refer to Terraform Enterprise's self-managed policy for its [network security](/terraform/enterprise/deploy/replicated/architecture/system-overview/security-model#terraform-enterprise-requires-you-to-manage-and-secure-the-underlying-network-and-infrastructure).

## Create a Terraform Enterprise team

We suggest you create a new Terraform Enterprise team to use for your Infragraph connection. To give Infragraph access to all workspaces in your Terraform organization, give the team the following permissions:

- View all projects
- View all workspaces

You can use Terraform Enterprise permissions to limit the scope of your API token to specific projects or workspaces. Refer to the [Manage teams](/terraform/enterprise/users-teams-organizations/teams/manage#manage-workspace-access) documentation for more information.

## Create a Terraform Enterprise team token

To connect Infragraph to Terraform Enterprise, you must provide an API token. Refer to the [Team API Tokens](/terraform/enterprise/users-teams-organizations/api-tokens#team-api-tokens) documentation for more information.

## Set up the connection 

To establish a new Terraform Enterprise connection:

1. From the Resource graph **Overview**, at the top right, click **Create Connection**.
1. Select **Terraform Enterprise** from the list of data sources.
1. Click **Select Terraform**.
1. Provide the hostname of the Terraform Enterprise instance. For example, `terraform.example.com`.
1. Follow the instructions to paste the team token. 
1. Click **Create connection**.

The data source connects to your Terraform Enterprise organization.

## Sync Terraform Enterprise data

To find the current sync status for your Terraform Enterprise data:

1. Click **Connections** in your HCP organization's left navigation panel.
1. Check the Terraform Enterprise connection sync status.

    - `Awaiting initial sync` means that the connection was created but the first data sync has not yet initiated.
    - `Successful` means there were no errors in the latest sync and the data was synced. 
    - `Partial failure` means at least one error occurred in the latest sync. 

To manually initiate a new sync, next to the data source, under **Actions**, click **`…`** and then select **Sync connection**.

## View Terraform Enterprise data

To find Terraform Enterprise data connected to your resource graph, try any of the following actions:

- Go to the **Inventory** page. Click an individual resource type to view it in the graph explorer. 
- Query resources with any of the existing queries that include Terraform Enterprise resources.
- Build the query from scratch using Terraform Enterprise resources in the query editor. For easy filtering, you can select **IaC** or **Management** options.
