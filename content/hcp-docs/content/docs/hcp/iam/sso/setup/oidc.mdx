# Set up OIDC for SSO

This page describes the process to set up Okta integration for HCP single sign-on. You can configure HCP for OIDC SSO with the following identity providers:

- Auth0
- AWS
- Azure Entra ID
- CyberArk
- Duo Security
- Google Cloud
- JumpCloud
- Okta
- One Login
- PingID

## Initiate integration on HCP

1. [Log in to HCP](https://portal.cloud.hashicorp.com/) and go to your organization.
1. From your organization, click **Organization settings**.
1. Click **SSO**. Then click **Configure SSO for your organization**.
1. Select **OIDC**.
1. Enter the following values from your configured identity provider:

  - **Client ID**
  - **Client Secret**
  - **Issuer URL**

## Complete SSO setup

1. Assign a [default organization role](/hcp/docs/hcp/iam/sso/default-role) for users.
1. Optionally, turn on **Assign users an organization role**.
1. Click **Save**.

Now users can sign in to your HCP organization using an existing identity provider.
