# HCP CLI overview for HCP Vault Secrets

The HCP CLI supports HCP Vault Secrets using the `vault-secrets` command group.

Refer to the [HCP CLI section](/hcp/docs/cli/commands/vault-secrets) for a
complete list of all supported `vault-secrets` commands.

## Login to HCP Vault Secrets

You can login interactively using a web browser or configure the `hcp`
CLI for non-interactive workflows.

<Tabs>
<Tab heading="Interactive">

The `hcp auth login` command will authenticate you with the HashiCorp Cloud Platform.

1. Open a terminal and run `hcp auth login`.

   ```shell-session
   $ hcp auth login
   ```

   A new browser window will open.

1. Log into [the HCP Portal](https://portal.cloud.hashicorp.com/) using your username and password.

1. Return to the terminal, the `Successfully logged in!` message appears.

   The `auth login` command will try to configure default organization and
   project. When multiple organizations or projects are detected, you will be
   prompted to run `hcp profile init`.

1. Run `hcp profile init` to configure (or update) the default profile.

   ```shell-session

   ```

<Tab heading="Non-interactive">

Use the `hcp profile set` command to configure the CLI to authenticate with the
desired HCP organization, project, and HCP Vault secrets app using an HCP
service principal. With the values set, the `hcp` CLI can work with HCP Vault
Secrets without human interaction.

1. Create a [service principal](/hcp/docs/hcp/iam/service-principal) and
   [key](/hcp/docs/hcp/iam/service-principal/key) for the service principal.

1. Set the `HCP_CLIENT_ID` and `HCP_CLIENT_SECRET` environment variables with
   values from the previous step.

   ```shell-session
   $ export HCP_CLIENT_ID=<actual-client-id> HCP_CLIENT_SECRET=<actual-client-secret>
   ```

1. Collect the organization ID, project ID, and app name you want to use with
   the `hcp` CLI.

1. Add the organization ID to the default `hcp` CLI profile.

   ```shell-session
   $ hcp profile set organization_id <org-id> --quiet
   ```

1. Add the organization ID to the default `hcp` CLI profile.

   ```shell-session
   $ hcp profile set project_id <project-id> --quiet
   ```

1. Add the organization ID to the default `hcp` CLI profile.

   ```shell-session
   $ hcp profile set vault-secrets/app <app-name> --quiet
   ```

1. You can now use the `hcp` CLI in an automated manner.

   ```shell-session
   $ hcp vault-secrets open <secret-name>
   ```

## Create a secret

Use the `vault-secrets` command family to create secrets.

You can create a secret by specifying a data file, or piping the secret using
STDIN.

   ```shell-session
   $ echo -n "WorstPasswordEver" | hcp vault-secrets secrets create password --data-file=-
   Secret Name   Created At                 Latest Version
   password      2024-10-04T16:35:14.958Z   1

   ✓ Successfully created secret with name "password"

   To read your secret, run:
     $ hcp vault-secrets secrets read password --app ExampleApplication
   ```

## Read a secret

Use the `vault-secrets` command family to open secrets.

The `open` command returns the value of a secret.

   ```shell-session
   $ hcp vault-secrets secrets open password
   Secret Name:    password
   Type:           kv
   Created At:     2024-10-04T16:35:14.958Z
   Latest Version: 1
   Value:          WorstPasswordEver
   ```

## Delete a secret

Use the `vault-secrets` command family to delete secrets.

The `delete` command to remove a secret.

   ```shell-session
   $ hcp vault-secrets secrets  delete password
   ✓ Successfully deleted secret with name "password"
   ```

## Logout of HCP Vault Secrets

Use the `auth` command family to logout.

   ```shell-session
   $ hcp auth logout
   Successfully logged out
   ```
