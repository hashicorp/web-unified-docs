# Update static secrets

Static secrets are created as key/value pairs. Static secrets typically persist
over long periods of time.

Refer to the [Design a secret strategy
documentation](/hcp/docs/vault-secrets/get-started/plan-implementation/design-strategry)
for information and tips on app and secret conventions.

## Prerequisites

- An HCP IAM user or service principal with the [HCP contributor
  role](/hcp/docs/vault-secrets/get-started/configure-user-permissions) or
  higher.
- An HCP Vault Secrets [app
  created](/hcp/docs/vault-secrets/get-started/create-app).
- Authenticated to the [HCP Portal](/hcp/docs/vault-secrets/get-started/access-ui) or [HCP CLI](/hcp/docs/cli/commands/auth/login)

<Tabs>
<Tab heading="CLI">

static secret.

@include 'hcp-cli/commands/vault-secrets/create-examples.mdx'

<Tab heading="UI">

1. From the HCP Vault Secrets dashboard, select the app you want to create a
   secret in.

1. Click the ellipses to the right of the secret and select **Edit secret
   value**.

1. Enter a new secret in the **Value** textbox.

1. Click **Save**.
