API gateway reconciliation

Starting with Consul 2.0, Consul reconciles only API gateways that are controlled by Consul and contain the required labels in the API gateway object. Consul validates the object at two levels: the object must contain the appropriate label and the controller name of the object must match.

Standard gateway

For the standard Kubernetes Gateway API, use the following labels:

apiVersion: gateway.networking.k8s.io/v1
kind: Gateway
metadata:
  name: api-gateway # can be any
  namespace: consul
  labels:
    component: api-gateway # should be this

Custom Consul gateway

For the custom Consul Gateway resource, use the following labels:

apiVersion: consul.hashicorp.com/v1beta1
kind: Gateway
metadata:
  name: api-gateway-consul # can be any
  namespace: consul
  labels:
    component: api-gateway-consul # should be this
